# 🏭 流计算工业案例库

> 真实生产环境中的流计算应用案例集合

## 📖 简介

本目录收集并整理了来自各行业生产环境的流计算应用案例。这些案例展示了 Flink、Kafka Streams、Spark Streaming 等流计算技术在实际业务场景中的应用，涵盖架构设计、性能优化、问题排查等宝贵经验。

## 📝 如何提交案例

### 方式一：通过 GitHub Issue 提交（推荐）

1. 点击 [New Issue](../../issues/new/choose)
2. 选择 **🏭 工业案例征集** 模板
3. 按照模板填写详细信息
4. 提交 Issue

### 方式二：直接提交 PR

1. Fork 本仓库
2. 在 `case-studies/submitted/` 目录下创建案例文件
3. 参考 [template.md](./template.md) 格式编写
4. 提交 Pull Request

### 提交要求

- [ ] 使用提供的模板格式
- [ ] 包含完整的六段式结构（定义、属性、关系、论证、证明、实例）
- [ ] 提供系统架构图（Mermaid 或图片）
- [ ] 包含量化的数据规模和性能指标
- [ ] 说明隐私处理情况和授权范围
- [ ] 不包含敏感信息和商业机密

## ✅ 审核流程

```
提交 → 初步审核 → 反馈修改 → 技术审核 → 隐私确认 → 收录发布
  ↑___________________________________|
```

1. **初步审核** (1-3天) - 检查格式完整性和基本信息
2. **反馈修改** - 如有问题，维护者将提出修改建议
3. **技术审核** (3-5天) - 技术专家验证技术内容的准确性
4. **隐私确认** - 确认敏感信息已脱敏，授权声明完整
5. **收录发布** - 案例移动到 `verified/` 目录并在主页展示

## 🔒 隐私保护

### 我们的承诺

- 所有案例在发布前均经过敏感信息审查
- 支持多级匿名选项（完全公开/半匿名/完全匿名）
- 未经明确授权，不会公开公司名称和联系方式
- 可随时申请更新或撤下已发布的案例

### 提交者须知

- 请务必脱敏内部主机名、IP地址、项目代号
- 业务指标可以用百分比或相对值代替绝对值
- 如不确定是否敏感，建议先咨询公司法务
