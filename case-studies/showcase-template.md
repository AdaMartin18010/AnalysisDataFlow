## 🎨 视觉设计规范

### 颜色方案

| 元素 | 颜色 | 色值 | 用途 |
|------|------|------|------|
| 主色调 | 深蓝 | `#1565C0` | 标题、重点 |
| 辅色调 | 青色 | `#00838F` | 副标题、标签 |
| 强调色 | 橙色 | `#F57C00` | 关键数据、CTA |
| 成功色 | 绿色 | `#2E7D32` | 正向指标 |
| 背景色 | 浅灰 | `#F5F5F5` | 卡片背景 |

### 排版规范

| 元素 | 字体大小 | 字重 | 样式 |
|------|----------|------|------|
| 案例标题 | 32px | Bold | - |
| 公司名称 | 24px | Medium | - |
| 章节标题 | 20px | SemiBold | 左侧边框3px |
| 正文 | 16px | Regular | 行高1.6 |
| 标注 | 14px | Regular | 灰色 #666 |

## 🏗️ 架构图模板

### 标准架构图格式

```markdown
### 系统架构图

```mermaid
graph TB
    subgraph "数据来源层"
        A1[数据源A]
        A2[数据源B]
        A3[数据源C]
    end

    subgraph "消息队列层"
        B1[Kafka Topic A]
        B2[Kafka Topic B]
    end

    subgraph "流计算层"
        C1[Flink Job 1]
        C2[Flink Job 2]
        C3[Flink Job 3]
    end

    subgraph "存储层"
        D1[实时存储<br/>Redis/ClickHouse]
        D2[离线存储<br/>HDFS/S3]
    end

    subgraph "应用层"
        E1[实时大屏]
        E2[告警系统]
        E3[数据服务]
    end

    A1 --> B1
    A2 --> B1
    A3 --> B2

    B1 --> C1
    B2 --> C2
    C1 --> C3
    C2 --> C3

    C3 --> D1
    C3 --> D2

    D1 --> E1
    D1 --> E2
    D2 --> E3

    style C1 fill:#BBDEFB,stroke:#1565C0,stroke-width:2px
    style C2 fill:#BBDEFB,stroke:#1565C0,stroke-width:2px
    style C3 fill:#FFCCBC,stroke:#E64A19,stroke-width:3px
    style D1 fill:#C8E6C9,stroke:#2E7D32
```

## 📱 多平台适配

### GitHub 展示

- 使用 GitHub 支持的 Mermaid 图表
- 使用 Emoji 增强可读性
- 保持表格在 GitHub 预览下的格式

### 网站展示

- 支持响应式布局
- 支持深色/浅色模式切换
- 架构图支持缩放交互

### PDF 导出

- 固定布局确保打印效果
- 图片使用高分辨率版本
- 分页控制避免内容截断
