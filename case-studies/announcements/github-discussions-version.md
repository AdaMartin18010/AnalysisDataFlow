> **征集活动**: AnalysisDataFlow 工业案例征集 2026
> **征集周期**: 2026年4月15日 - 2026年12月31日
> **主办单位**: AnalysisDataFlow 项目团队

## 📢 活动背景

流计算技术（Apache Flink、Kafka Streams、Spark Streaming 等）已成为现代数据架构的核心组件，广泛应用于实时推荐、风控反欺诈、IoT 数据处理等场景。

然而，**来自真实生产环境的高质量案例仍然相对稀缺**。很多优秀的工程实践沉淀在企业内部，未能惠及更广泛的开发者社区。

为此，我们正式启动 **2026 流计算工业案例征集活动**，旨在建立一个**开源、权威、可持续更新**的流计算工业实践知识库。

## ✅ 案例要求

### 基本要求

- ✅ 来自**真实生产环境**（非 POC/测试环境）
- ✅ 包含**量化的性能指标**（吞吐量、延迟、数据规模）
- ✅ 提供**系统架构图**或技术方案
- ✅ 分享有实际价值的**经验教训**
- ✅ **不包含敏感信息**和商业机密

### 完整案例结构

一个完整的工业案例应包含以下 8 个部分：

```
1. 基本信息        - 公司、行业、联系人
2. 使用场景        - 业务背景、核心价值
3. 数据规模        - 吞吐量、集群规模、延迟要求
4. 架构图          - 系统架构、技术栈、数据流
5. 经验总结        - 技术选型、挑战与解决方案
6. 项目成果        - 量化指标、业务价值
7. 隐私与授权      - 敏感信息处理、授权声明
8. 附加材料        - 文档链接、代码片段
```

## 🔒 隐私保护承诺

### 三级匿名选项

| 级别 | 说明 | 适用场景 |
|:----:|:-----|:---------|
| 🔓 完全公开 | 可公开公司名称、Logo、所有技术细节 | 公司允许公开技术分享 |
| 🔐 半匿名 | 公开行业类型（如"某电商平台"），隐藏公司名 | 可分享技术但需匿名 |
| 🔒 完全匿名 | 仅展示技术架构，不透露任何公司信息 | 信息管控严格的公司 |

### 保护措施

1. **敏感信息审查** - 所有案例发布前经过脱敏审查
2. **授权确认** - 未经明确授权不公开公司信息
3. **随时撤销** - 可随时申请更新或撤下已发布内容
4. **安全存储** - 案例源码安全存储，仅维护团队可访问

## ❓ 常见问题（FAQ）

**Q1: 公司内部信息管控严格，可以参与吗？**
> 完全可以。我们支持**完全匿名提交**，您只需分享技术方案和架构，无需透露任何公司信息。所有敏感信息都可以脱敏处理。

**Q2: 案例发布后，可以修改或撤下吗？**
> 可以。您随时可以申请更新或撤下已发布内容，我们将在 **2 个工作日内**处理。

**Q3: 案例审核不通过怎么办？**
> 审核团队会提供具体的修改建议，您可以完善后重新提交。常见不通过原因包括：缺少关键信息、敏感信息未脱敏、内容不够详细。

**Q4: 提交案例有截止日期吗？**
> 本次征集活动长期有效，但建议在 **2026年12月31日** 前提交，以便收录进 2026 年度案例集。

**Q5: 一个公司可以提交多个案例吗？**
> 当然可以。我们鼓励分享不同场景、不同技术栈的多个案例。

**Q6: 个人可以提交案例吗？**
> 可以。只要案例来自真实生产环境，个人贡献者同样欢迎。

**Q7: 案例需要英文版吗？**
> 中文即可。如有英文版更好，便于国际社区阅读，但不是必须。

**Q8: 如何跟踪审核进度？**
> 通过 GitHub Issue 提交的案例，可在对应 Issue 中查看审核进度和反馈。邮件提交的案例，我们会邮件通知审核结果。
