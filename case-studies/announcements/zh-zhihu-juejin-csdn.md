# 🏭 【案例征集】流计算工业实践案例 - 共建权威知识库

> 征集来自真实生产环境的流计算应用案例，建立流计算领域最全面的工业实践知识库

---

## 📊 项目背景

流计算技术（Flink、Kafka Streams、Spark Streaming 等）在工业界的应用越来越广泛，从实时推荐到风控反欺诈，从 IoT 数据处理到实时大屏，流计算已成为现代数据架构的核心组件。

然而，**高质量的工业案例分享仍然相对稀缺**。很多优秀的生产实践沉淀在各公司的内部文档中，未能惠及更广泛的开发者社区。

我们正在建设一个**开源、权威、可持续更新**的流计算工业案例库，诚邀各位技术同仁贡献宝贵的实践经验。

---

## 🎯 征集范围

### 技术栈

| 技术 | 说明 |
|------|------|
| Apache Flink | 重点征集，包括 DataStream API、Table API、SQL |
| Kafka Streams | 轻量级流处理，适合微服务场景 |
| Spark Streaming | Structured Streaming 生产实践 |
| 其他 | Storm、Pulsar Functions、Beam 等 |

### 场景类型

- 🔐 **实时风控/反欺诈** - 交易风控、账号安全、行为分析
- 🎯 **实时推荐系统** - 内容推荐、商品推荐、个性化推送
- 📈 **实时数据分析** - 实时大屏、业务指标监控、用户行为分析
- 🌐 **IoT 实时处理** - 设备数据采集、边缘计算、实时监控
- 🔄 **实时 ETL/数据管道** - 数据同步、格式转换、数据清洗
- 💡 **其他创新应用** - 任何有趣的流计算实践

## ✅ 案例要求

我们寻找来自**真实生产环境**的案例：

- ✅ 已在线上环境稳定运行的系统
- ✅ 包含量化的性能指标（吞吐量、延迟、数据规模）
- ✅ 提供系统架构图或技术方案
- ✅ 分享有实际价值的经验教训
- ✅ 不包含敏感信息和商业机密

---

## 💡 案例价值

优秀的案例将包含：

1. **系统架构图** - 展示整体技术架构和数据流
2. **数据规模** - 吞吐量(QPS)、集群规模、数据量(TB/PB)
3. **性能指标** - 延迟分布、资源利用率、可用性 SLA
4. **技术选型** - 为什么选择这个技术方案
5. **挑战与解决** - 踩过的坑和解决方案
6. **最佳实践** - 总结可复用的经验和模式

## 🔒 隐私保护承诺

我们深知生产环境信息的敏感性：

| 保护级别 | 说明 |
|----------|------|
| 🔓 完全公开 | 可公开公司名称、Logo、所有技术细节 |
| 🔐 半匿名 | 公开行业类型（如：某电商平台），隐藏公司具体名称 |
| 🔒 完全匿名 | 仅展示技术架构，不透露任何公司信息 |

**其他保护措施：**
- 案例发布前经过敏感信息审查
- 未经明确授权不公开公司信息
- 可随时申请更新或撤下已发布内容

## 📋 案例模板预览

完整案例包含以下 8 个部分：

1. **基本信息** - 公司、行业、技术栈
2. **使用场景** - 业务背景、核心价值
3. **数据规模** - 吞吐量、集群规模、延迟要求
4. **架构图** - 系统架构、技术栈、数据流
5. **经验总结** - 技术选型、挑战与解决方案
6. **项目成果** - 量化指标、业务价值
7. **隐私与授权** - 敏感信息处理、授权声明
8. **附加材料** - 文档链接、代码片段
