### 规约结构

- CONSTANT 声明（可配置参数）
- VARIABLE 声明（状态变量）
- Init 谓词（初始状态）
- Next 谓词（状态转移）
- Spec 时序公式（完整规约）
- Invariant 不变式（安全性）
- Temporal 活性公式（Liveness）

  Value = {v1, v2}
  
INVARIANTS
  TypeInvariant

PROPERTIES
  LivenessProperty

## 📋 主线6: 分析论证深化（251篇）

### 深化内容清单

对每篇现有文档，增加以下部分：

#### 6.1 完整证明链

**要求**: 从公理出发，无跳步地推导到结论

**模板**:
```markdown
**证明**:
1. 基础情况：...
   - 根据Def-X-XX，...
   - 由Lemma-X-XX，...
   
2. 归纳步骤：假设...，证明...
   - 由归纳假设，...
   - 应用Thm-X-XX，...
   
3. 结论：因此，QED。
```

#### 6.2 反例构造

**要求**: 对每个"如果"条件，构造违反条件的反例

**模板**:
```markdown
**反例 X.1** (当条件Y不成立时):
- 构造：...
- 分析：此时定理不成立，因为...
- 图示：...
```

#### 6.3 复杂度分析

**要求**: 算法时间/空间复杂度的完整分析

**模板**:
```markdown
**复杂度分析**:
- **时间复杂度**: T(n) = O(...)
  - 最坏情况：...
  - 平均情况：...
  
- **空间复杂度**: S(n) = O(...)
  - 栈深度：...
  - 堆分配：...
  
- **下界**: Ω(...)证明
  - 信息论下界：...
```

#### 6.4 等价性证明

**要求**: 证明不同定义的等价性

**模板**:
```markdown
**等价性定理** (Thm-X-XX):
定义A与定义B等价，即 A ⇔ B

**证明**:
- (⇒) 假设定义A成立，证明定义B：...
- (⇐) 假设定义B成立，证明定义A：...
```
