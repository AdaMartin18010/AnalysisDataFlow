## 1. 概念定义 (Definitions)

### Def-S-05-01 (CSP 核心语法)

CSP 核心语法 [^1][^2]：

$$
\begin{aligned}
P, Q ::= &\ \text{STOP} \mid \text{SKIP} \mid a \to P \mid P \mathbin{\square} Q \mid P \mathbin{\sqcap} Q \\
       &\mid P \mathbin{|||} Q \mid P \mathbin{\parallel_A} Q \mid P \setminus A \mid P; Q \mid \mu X.F(X)
\end{aligned}
$$

其中 $\Sigma$ 为全局事件字母表，$\tau \notin \Sigma$ 为内部动作，$\checkmark \in \Sigma$ 为成功终止事件。外部选择 $\square$ 由环境决定分支，内部选择 $\sqcap$ 体现进程自主非确定性。

**直观解释**：CSP 通过显式事件组合并发进程，同步并行 $\parallel_A$ 精确控制哪些事件需要多方同时就绪，隐藏算子 $\setminus A$ 支持模块化抽象。

**定义动机**：静态命名与组合算子使 FDR4 等工具能够对有限状态子集进行穷尽验证，而 STOP/SKIP 的区分支撑了精化关系与多层级语义模型 [^3]。

### Def-S-05-02 (CSP 结构化操作语义)

核心 SOS 规则 [^2][^3]：

```
              a → P ─a→ P                           [Prefix]

              P ─a→ P'
[Ext-L] ────────────────────
              P □ Q ─a→ P'

              Q ─a→ Q'
[Ext-R] ────────────────────
              P □ Q ─a→ Q'

              P ─τ→ P'
[Int-L] ────────────────────
              P ⊓ Q ─τ→ P'

              Q ─τ→ Q'
[Int-R] ────────────────────
              P ⊓ Q ─τ→ Q'

              P ─a→ P'
[Inter-L] ─────────────────────────
              P ||| Q ─a→ P' ||| Q

              Q ─a→ Q'
[Inter-R] ─────────────────────────
              P ||| Q ─a→ P ||| Q'

              P ─a→ P'        Q ─a→ Q'
[Sync] ───────────────────────────────────────
              P |[A]| Q ─a→ P' |[A]| Q'    (a ∈ A)

              P ─a→ P'        a ∉ A
[Sync-L] ───────────────────────────────────────
              P |[A]| Q ─a→ P' |[A]| Q

              Q ─a→ Q'        a ∉ A
[Sync-R] ───────────────────────────────────────
              P |[A]| Q ─a→ P |[A]| Q'

              P ─a→ P'        a ∈ A
[Hide] ───────────────────────────────────────
              P \ A ─τ→ P' \ A

              P ─a→ P'        a ∉ A
[Hide-Vis] ─────────────────────────────────────
              P \ A ─a→ P' \ A

              P ─a→ P'        a ≠ ✓
[Seq-L] ───────────────────────────────────────
              P ; Q ─a→ P' ; Q

              P ─✓→ P'
[Seq-R] ───────────────────────────────────────
              P ; Q ─τ→ Q

              F[μX.F/X] ─a→ P'
[Rec] ───────────────────────────────────────
              μX.F ─a→ P'
```

**直观解释**：SOS 将 CSP 语法转化为标记转移系统。规则 **[Sync]** 要求同步集 $A$ 内的事件由多方同时参与，实现多路同步握手；**[Hide]** 将指定事件内部化为不可见的 $\tau$。

**定义动机**：SOS 为迹语义、失败语义和互模拟等价提供了可直接验证的转移关系基础，是 CSP 工业级模型检验的理论根基 [^3]。

### Def-S-05-03 (CSP 迹语义、失败语义与发散语义)

#### 迹语义 (Trace Semantics)

进程 $P$ 的迹集合 $\text{traces}(P) \subseteq (\Sigma \cup \{\checkmark\})^*$ 归纳定义如下 [^2][^3]：

$$
\begin{aligned}
\text{traces}(\text{STOP}) &= \{\varepsilon\} \\
\text{traces}(\text{SKIP}) &= \{\varepsilon, \langle \checkmark \rangle\} \\
\text{traces}(a \to P) &= \{\varepsilon\} \cup \{a \cdot s \mid s \in \text{traces}(P)\} \\
\text{traces}(P \mathbin{\square} Q) &= \text{traces}(P) \cup \text{traces}(Q) \\
\text{traces}(P \mathbin{\sqcap} Q) &= \text{traces}(P) \cup \text{traces}(Q) \\
\text{traces}(P \mathbin{|||} Q) &= \{\text{interleave}(s, t) \mid s \in \text{traces}(P), t \in \text{traces}(Q)\} \\
\text{traces}(P \mathbin{\parallel_A} Q) &= \{s \mid s \restriction A \in \text{traces}(P) \restriction A \cap \text{traces}(Q) \restriction A,\ s \restriction (\Sigma \setminus A) \in \text{interleave}(\dots)\} \\
\text{traces}(P \setminus A) &= \{s \restriction (\Sigma \setminus A) \mid s \in \text{traces}(P)\} \\
\text{traces}(P; Q) &= \text{traces}(P) \mathbin{;\!\!_\checkmark} \text{traces}(Q) \\
\text{traces}(\mu X.F) &= \bigcup_{n \geq 0} \text{traces}(F^n[\text{STOP}/X])
\end{aligned}
$$

#### 失败语义 (Failures Semantics)

失败集合定义为 [^3]：
$$
\text{failures}(P) = \{(s, X) \mid s \in \text{traces}(P), P \text{ after } s \text{ can refuse } X\}
$$
迹语义无法区分 $\square$ 与 $\sqcap$，但失败语义可以：例如 $a \to \text{STOP} \mathbin{\square} b \to \text{STOP}$ 与 $a \to \text{STOP} \mathbin{\sqcap} b \to \text{STOP}$ 迹相同，但失败集不同。

#### 发散语义 (Divergences Semantics)

发散集合定义为 [^3]：
$$
\text{divergences}(P) = \{s \cdot t \mid s \in \text{traces}(P), \exists Q \in (P \text{ after } s). Q \xrightarrow{\tau^\omega}\}
$$
隐藏算子 $\setminus A$ 可引入发散：若 $P$ 在 $A$ 上无限循环，则 $P \setminus A$ 陷入不可见的无限 $\tau$ 序列（活锁）。

**直观解释**：迹语义刻画“能做什么”，失败语义进一步刻画“可能拒绝什么”，发散语义刻画“是否陷入不可见无限循环”。三者由粗到细支撑了 CSP 的 $T$、$F$、$FD$ 精化关系。

**定义动机**：失败语义区分外部/内部选择，是验证环境可控性的关键；发散语义则检测活锁——工业并发系统中比死锁更隐蔽的缺陷 [^3]。

### Def-S-05-04 (CSP 通道与同步原语)

CSP 的通道事件字母表定义为 $\Sigma = \bigcup_{c \in \mathcal{C}} \{c.v \mid v \in \mathcal{V}_c\} \cup \{\checkmark\}$。引入语法糖 [^2][^3]：

- **输入前缀**：$c?x \to P \equiv \square_{v} (c.v \to P[v/x])$
- **输出前缀**：$c!v \to P \equiv c.v \to P$
- **同步事件**：$c.v$ 为发送方与接收方共同参与的单一原子事件。

**多路同步**：CSP 的 $\parallel_A$ 支持任意多个进程在同一事件上同步（如屏障同步），而不仅限于二元握手。通道方向性通过限制进程的 alphabet 实现：仅含输出事件的 alphabet 对应发送端，仅含输入事件对应接收端。

**直观解释**：CSP 通道是同步握手点而非消息队列，通信事件 $c.v$ 是瞬时、原子的。这种设计消除了异步缓冲的复杂性，使模型检验更可行；但异步语义需引入辅助缓冲进程 [^3]。

**定义动机**：将 $c.v$ 建模为单一同步事件，是证明 Go 无缓冲 channel 与 CSP 迹等价（Thm-S-05-01）的关键桥梁；多路同步则扩展了 CSP 对分布式协议的建模能力 [^2]。

### Def-S-05-05 (Go-CS-sync 到 CSP 的编码函数)

Go-CS-sync（仅无缓冲 channel 的 Go 并发子集）到 CSP 的编码 $[\![ \cdot ]\!]$ 定义如下 [^6][^7]：

| Go 构造 | CSP 编码 |
|---------|----------|
| $0$ | $\text{STOP}$ |
| $\text{go}\ P$ | $[\![P]\!] \mathbin{|||} \text{SKIP}$ |
| $ch \leftarrow v; P$ | $ch.v \to [\![P]\!]$ |
| $x := \leftarrow ch; P$ | $ch?x \to [\![P]\!]$ |
| $P; Q$ | $[\![P]\!]; [\![Q]\!]$ |
| $\text{select}\ \{\text{case}_i : P_i\}$ | $\square_i\ (a_i \to [\![P_i]\!])$ |
| $\text{rec}\ X.P$ | $\mu X.[\![P]\!]$ |

其中 $a_i$ 为第 $i$ 个分支的首事件（发送为 $ch_i.v_i$，接收为 $ch_i?x_i$）。

**直观解释**：`go` 映射为交错并行，channel 操作映射为同步事件前缀，`select` 映射为外部选择，递归映射为 $\mu$ 算子。

**定义动机**：该编码将 Go-CS-sync 的分析归约到 CSP 理论框架，使得可以应用 FDR4 等工具验证死锁自由与精化关系，同时清晰标识 Go 与 CSP 之间的语义边界 [^3][^7]。

## 2. 属性推导 (Properties)

### Property 1 (CSP 前缀的迹前缀闭包性)

**陈述**：$\text{traces}(a \to P)$ 是前缀闭包的。

**推导**：

1. 由 Def-S-05-03，$\text{traces}(a \to P) = \{\varepsilon\} \cup \{a \cdot s \mid s \in \text{traces}(P)\}$。
2. 对任意 $s \in \text{traces}(P)$，$a \cdot s$ 的前缀为 $\varepsilon$ 或 $a \cdot s'$（$s'$ 为 $s$ 的前缀）。
3. 迹语义本身保证前缀闭包，故 $s' \in \text{traces}(P)$，从而 $a \cdot s' \in \text{traces}(a \to P)$。
4. 得证。 ∎

### Property 2 (CSP 交错并行的结合律与交换律)

**陈述**：$P \mathbin{|||} (Q \mathbin{|||} R) = (P \mathbin{|||} Q) \mathbin{|||} R$ 且 $P \mathbin{|||} Q = Q \mathbin{|||} P$。

**推导**：

1. 由 Def-S-05-03，$\text{traces}(P \mathbin{|||} Q) = \{\text{interleave}(s,t)\}$。
2. 序列交错运算满足结合律与交换律（保持各子序列内部顺序，子序列间无优先级）。
3. 因此迹集合层面的结合律与交换律成立。 ∎

### Property 3 (CSP 外部选择可执行分支集合单调性)

**陈述**：$\text{ready}(P \mathbin{\square} Q) = \text{ready}(P) \cup \text{ready}(Q)$。

**推导**：

1. 由 SOS 规则 **[Ext-L]** 和 **[Ext-R]**，$P \mathbin{\square} Q$ 的初始转移只能来自 $P$ 或 $Q$。
2. 外部选择没有内部 $\tau$ 规则，因此初始可执行事件集恰好是两子进程的并集。
3. 得证。 ∎

### Property 4 (CSP 同步并行在空同步集上的退化)

**陈述**：$P \mathbin{\parallel_\emptyset} Q = P \mathbin{|||} Q$。

**推导**：

1. 当 $A = \emptyset$ 时，$s \restriction A = \varepsilon$，同步条件 trivially 成立。
2. 迹语义退化为 $s \in \text{interleave}(\text{traces}(P), \text{traces}(Q))$，即交错并行语义。 ∎

### Property 5 (Go 无缓冲 channel 通信与 CSP 同步事件的原子性等价)

**陈述**：Go-CS-sync 中无缓冲 channel 的成功通信对应于单一内部 $\tau$ 转移，在 CSP 中对应单一同步事件 $ch.v$。

**推导**：

1. Go 的无缓冲发送/接收必须双方同时就绪，运行时将其归约为一步内部 $\tau$ 转移 [^6]。
2. CSP 编码将发送/接收映射为同一事件 $ch.v$，在 $\parallel_{\{ch.v\}}$ 下要求双方同时参与（规则 **[Sync]**）。
3. 因此两者在原子性与同时性约束上等价。 ∎

## 3. 关系建立 (Relations)

### 关系 1: CSP 与 CCS 的语义不可比较性

CSP 基于迹/失败/发散语义，CCS 基于互模拟语义。强互模拟细于迹等价，但与失败等价不可比；CSP 的 STOP/SKIP 区分在 CCS 中无法精确保留。因此 CSP $\perp$ CCS（语义不可比较）。详见 [01.02-process-calculus-primer.md](./01.02-process-calculus-primer.md) 中关系 1 [^4][^5]。

### 关系 2: CSP 与 π-演算的表达能力包含关系

存在从 CSP 到 π-演算的迹保持编码，但 π-演算支持运行时名字创建 $(\nu a)$ 与名字传递，CSP 的静态命名无法模拟动态拓扑变化。因此 **CSP $\subset$ π-演算**（严格弱于），对应表达能力层次 L₃ $\subset$ L₄。详见 [01.02-process-calculus-primer.md](./01.02-process-calculus-primer.md) 中 Def-S-02-03 与 Thm-S-02-01 [^4][^5]。

### 关系 3: Go-CS-sync 与 CSP 的迹语义等价关系

由 Def-S-05-05 的编码函数，存在双向翻译：Go-CS-sync $\mapsto$ CSP 与 CSP $\mapsto$ Go-CS-sync。Thm-S-05-01 将严格证明两者在迹语义下等价。因此 **Go-CS-sync $\iff$ CSP**（L₃ 等价）[^7]。

### 关系 4: Go select 到 CSP 外部选择的语义映射

Go 的 `select` 依赖于环境中哪个 channel 操作就绪，与 CSP 外部选择 $\square$ 的“环境决定分支”语义一致。当多个 case 同时就绪时，Go 的伪随机选择从外部观察等价于非确定性。因此 `select` $\mapsto$ $\square$。若包含 `default`，则语义偏离为滑垒选择或内部选择混合 [^7]。

## 4. 论证过程 (Argumentation)

### Lemma-S-05-01 (CSP 外部选择可执行分支集合保持性)

**陈述**：设 $P = \square_{i \in I} (a_i \to P_i)$，则 $\text{ready}(P) = \bigcup_{i \in I} \{a_i\}$。

**证明**：

1. 由 SOS 规则 **[Ext-L]**/**[Ext-R]**，每个分支 $a_i \to P_i \xrightarrow{a_i} P_i$ 可提升为整体 $P \xrightarrow{a_i} P_i$。
2. 外部选择无内部 $\tau$ 规则，初始转移只能来自子分支。
3. 因此 $\text{ready}(P)$ 恰好是各分支首事件的并集。 ∎

### Lemma-S-05-02 (CSP 同步并行下迹前缀保持性)

**陈述**：若 $P \mathbin{\parallel_A} Q \xrightarrow{\alpha} R$，则 $\text{traces}(R) \subseteq \text{traces}(P \mathbin{\parallel_A} Q)$，且若 $\alpha \neq \tau$，则 $\alpha \cdot \text{traces}(R) \subseteq \text{traces}(P \mathbin{\parallel_A} Q)$。

**证明**：

1. **同步转移**（$\alpha \in A$）：由 **[Sync]**，$P \xrightarrow{\alpha} P'$，$Q \xrightarrow{\alpha} Q'$，$R = P' \mathbin{\parallel_A} Q'$。由于 $\text{traces}(P') \subseteq \text{traces}(P)$ 且同理对 $Q'$，迹集合包含性成立；对可见 $\alpha$，$\alpha \cdot \text{traces}(P') \subseteq \text{traces}(P)$ 且同理对 $Q'$，故 $\alpha \cdot \text{traces}(R) \subseteq \text{traces}(P \mathbin{\parallel_A} Q)$。
2. **独立转移**（$\alpha \notin A$）：不妨设 $P \xrightarrow{\alpha} P'$，$R = P' \mathbin{\parallel_A} Q$。迹投影到 $A$ 不变，投影到 $\Sigma \setminus A$ 增加 $\alpha$ 后仍在交错集合中。故结论成立。 ∎

### 论证: 迹等价定理的关键步骤

Thm-S-05-01 采用对 $Q \in \text{Go-CS-sync}$ 的结构归纳：

- **基础**：$Q = 0$ 映射为 STOP，迹均为 $\{\varepsilon\}$。
- **前缀**：发送/接收由 Property 5 保证同步事件 $ch.v$ 对应，结合归纳假设得证。
- **选择**：由 Lemma-S-05-01，外部选择的就绪分支集合与 `select` 一致，结合归纳假设得证。
- **并行**：`go P; R` 的交错与同步行为由编码 $[\![P]\!] \mathbin{\parallel_A} [\![R]\!]$ 捕获，Lemma-S-05-02 保证迹前缀保持性。
- **递归**：`rec` 与 $\mu$ 均定义为有限次展开的并集，归纳假设传递。

## 5. 形式证明 (Proofs)

### Thm-S-05-01 (Go-CS-sync 与 CSP 编码保持迹语义等价)

**陈述**：对于任意 $Q \in \text{Go-CS-sync}$，$\text{traces}_{\text{Go}}(Q) = \text{traces}_{\text{CSP}}([\![Q]\!])$。

**证明**：对 $Q$ 的结构归纳。

#### 基础情形 ($Q = 0$) {{#基础情形-q--0}}

$\text{traces}_{\text{Go}}(0) = \{\varepsilon\}$，$[\![0]\!] = \text{STOP}$，$\text{traces}_{\text{CSP}}(\text{STOP}) = \{\varepsilon\}$。✓

#### 归纳步骤

假设对 $Q$ 的所有真子进程成立。

**案例 1: 发送前缀 ($Q = ch \leftarrow v; P$)**

Go 迹：$\text{traces}_{\text{Go}}(Q) = \{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{Go}}(P)\}$。
编码：$[\![Q]\!] = ch.v \to [\![P]\!]$，其 CSP 迹为 $\{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P]\!])\}$。
由归纳假设，两者相等。✓

**案例 2: 接收前缀 ($Q = x := \leftarrow ch; P$)**

Go 在同步后执行 $P[x/v]$，迹为 $\{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{Go}}(P[x/v])\}$。
编码：$[\![Q]\!] = ch?x \to [\![P]\!]$，CSP 迹为 $\{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P]\!][v/x])\}$。
由归纳假设，两者相等。✓

**案例 3: 外部选择 ($Q = \text{select}\ \{\text{case}_i : P_i\}$)**

设分支 $i$ 的首事件为 $a_i$。Go 迹：$\{\varepsilon\} \cup \bigcup_i \{a_i \cdot s \mid s \in \text{traces}_{\text{Go}}(P_i)\}$。
编码：$[\![Q]\!] = \square_i (a_i \to [\![P_i]\!])$，CSP 迹：$\{\varepsilon\} \cup \bigcup_i \{a_i \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P_i]\!])\}$。
由归纳假设，两者相等。✓

**案例 4: 并行组合 ($Q = \text{go}\ P; R$)**

Go 中 `go P` 创建 goroutine 与 $R$ 交错执行，共享 channel 事件同步。设 $A$ 为 $P$ 与 $R$ 共享的 channel 事件集。
编码：$[\![Q]\!]$ 的迹语义等价于 $[\![P]\!] \mathbin{\parallel_A} [\![R]\!]$（因 $[\![P]\!] \mathbin{|||} \text{SKIP}$ 的可见迹等价于 $[\![P]\!]$）。
由 Def-S-05-03 的同步并行迹语义，该编码精确匹配 Go 的交错+同步行为。
由归纳假设，$\text{traces}_{\text{Go}}(P) = \text{traces}_{\text{CSP}}([\![P]\!])$ 且对 $R$ 同样成立，故整体相等。✓

**案例 5: 顺序组合 ($Q = P; R$)**

Go 迹为 $\text{traces}_{\text{Go}}(P)$ 与 $\text{traces}_{\text{Go}}(R)$ 的连接（Go 无显式 $\checkmark$）。
编码：$[\![P; R]\!] = [\![P]\!]; [\![R]\!]$，其 CSP 迹为 $\text{traces}_{\text{CSP}}([\![P]\!]) \mathbin{;\!\!_\checkmark} \text{traces}_{\text{CSP}}([\![R]\!])$。在去除 $\checkmark$ 后，与 Go 顺序组合迹一致。
由归纳假设得证。✓

**案例 6: 递归 ($Q = \text{rec}\ X.P$)**

Go 迹：$\bigcup_{n \geq 0} \text{traces}_{\text{Go}}(P^{(n)})$，其中 $P^{(0)} = 0$，$P^{(n+1)} = P[P^{(n)}/X]$。
编码：$[\![Q]\!] = \mu X.[\![P]\!]$，CSP 迹：$\bigcup_{n \geq 0} \text{traces}_{\text{CSP}}([\![P]\!]^{(n)})$（从 STOP 开始展开）。
由结构归纳，对每个 $n$，$\text{traces}_{\text{Go}}(P^{(n)}) = \text{traces}_{\text{CSP}}([\![P]\!]^{(n)})$。取并集后相等。✓

#### 结论

由结构归纳法，对所有 $Q \in \text{Go-CS-sync}$，$\text{traces}_{\text{Go}}(Q) = \text{traces}_{\text{CSP}}([\![Q]\!])$。 ∎

## 6. 实例验证 (Examples & Verification)

### 示例 1: Pipeline 模式的形式映射

Go 程序：

```go
func source(out chan int) {
    for i := 0; i < 3; i++ { out <- i }
    close(out)
}
func square(in, out chan int) {
    for x := range in { out <- x * x }
    close(out)
}
func main() {
    c1, c2 := make(chan int), make(chan int)
    go source(c1)
    go square(c1, c2)
    for x := range c2 { fmt.Println(x) }
}
```

CSP 编码：

```csp
SOURCE = c1.0 → c1.1 → c1.2 → c1.end → STOP
SQUARE = c1?x → c2.(x*x) → SQUARE □ c1.end → c2.end → STOP
CONSUMER = c2?x → print.x → CONSUMER □ c2.end → STOP
MAIN = (SOURCE ||| SQUARE) ||| CONSUMER
```

**验证**：`SOURCE` 的迹 $\langle c1.0, c1.1, c1.2, c1.end \rangle$ 经 `SQUARE` 变换后，与 `CONSUMER` 在同步并行下组合。由 Thm-S-05-01，该 CSP 模型与 Go 程序迹等价。

---

### 示例 2: 外部选择与 Go select 的 Vending Machine

CSP：

```csp
VM = coin → (tea → STOP □ coffee → STOP)
```

Go：

```go
func vm(coin, tea, coffee chan struct{}) {
    <-coin
    select {
    case <-tea:
    case <-coffee:
    }
}
```

**验证**：CSP 迹集合为 $\{\varepsilon, \langle coin \rangle, \langle coin, tea \rangle, \langle coin, coffee \rangle\}$。Go `select` 在 `tea` 或 `coffee` 就绪后选择对应分支，迹集合相同，验证了 `select` 到 $\square$ 的映射。

---

### 反例 1: 缓冲 channel FIFO 语义在纯 CSP 迹等价中的丢失

```go
ch := make(chan int, 2)
go func() { ch <- 1; ch <- 2 }()
go func() { <-ch; <-ch }()
```

Go 缓冲 channel 保证 FIFO，但 CSP 纯迹语义只记录事件出现与否，不记录缓冲队列的因果顺序。若用辅助进程 $Buffer(ch, 2)$ 编码，内部 $\tau$ 动作会破坏直接迹等价。因此缓冲 channel 的 FIFO 保证无法由纯 CSP 迹等价保持，表达能力需跃迁至 L₄ [^7]。

---

### 反例 2: CSP 隐藏算子的发散行为无法被 Go close(ch) 模拟

CSP 进程 $P = (a \to P) \setminus \{a\}$ 无限执行内部 $\tau$（发散）。Go 中 `close(a)` 只执行一次，即使尝试 `for { a <- struct{}{} }`，每次发送仍是可见同步事件，无法被隐藏为内部 $\tau$。因此 Go 缺少 CSP 隐藏算子的忠实对应物，不能引入发散 [^3][^7]。

---

### 反例 3: Go select with default 偏离 CSP 外部选择语义

```go
select {
case v := <-ch1: fmt.Println("ch1:", v)
case v := <-ch2: fmt.Println("ch2:", v)
default: fmt.Println("no data")
}
```

CSP 外部选择 $\square$ 会阻塞等待至少一个分支就绪，但 Go `select with default` 在所有 case 未就绪时立即执行 `default`，不等待。这在语义上更接近滑垒选择或内部选择混合，形式化分析时必须与纯 $\square$ 区分 [^7]。
