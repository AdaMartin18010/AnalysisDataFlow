## 1. 概念定义 (Definitions)

本节建立 CSP（Communicating Sequential Processes）的严格形式化基础，涵盖核心语法、结构化操作语义（SOS）、迹/失败/发散语义、通道同步原语，以及 Go-CS-sync 子集到 CSP 的编码函数。所有定义均为后续性质推导、关系建立与正确性证明的基石。

### Def-S-05-01 (CSP 核心语法)

CSP 是由 Hoare 于 1978 年提出、1985 年系统化的进程代数，其核心特征是通过**同步通信**与**静态事件名**组合并发系统 [^1][^2]。CSP 的核心语法定义如下：

$$
\begin{aligned}
P, Q ::= &\ \text{STOP} \quad \text{(死锁 — 拒绝所有事件)} \\
       |\ &\ \text{SKIP} \quad \text{(成功终止 — 可观测为特殊事件 } \checkmark \text{)} \\
       |\ &\ a \to P \quad \text{(前缀，事件 } a \in \Sigma \text{ 后执行 } P \text{)} \\
       |\ &\ P \mathbin{\square} Q \quad \text{(外部选择 — 环境决定分支)} \\
       |\ &\ P \mathbin{\sqcap} Q \quad \text{(内部选择 — 进程自主非确定性)} \\
       |\ &\ P \mathbin{|||} Q \quad \text{(交错并行 — 无同步约束)} \\
       |\ &\ P \mathbin{\parallel_A} Q \quad \text{(同步并行 — 在事件集 } A \subseteq \Sigma \text{ 上强制握手)} \\
       |\ &\ P \setminus A \quad \text{(隐藏 — 将 } A \text{ 中事件内部化为 } \tau \text{)} \\
       |\ &\ P; Q \quad \text{(顺序组合)} \\
       |\ &\ \mu X.F(X) \quad \text{(递归，最小不动点)}
\end{aligned}
$$

其中 $\Sigma$ 为全局事件字母表，$\tau \notin \Sigma$ 为内部不可观察动作，$\checkmark \in \Sigma$ 为成功终止事件。外部选择 $\square$ 与内部选择 $\sqcap$ 的区分是 CSP 区别于 CCS 的关键设计之一：前者将控制权交给环境（即外部通信就绪情况），后者体现进程内部的非确定性。

**直观解释**：CSP 将并发系统抽象为一组通过**显式事件**进行交互的进程。每个进程的行为由其可以参与的可见事件序列（迹）刻画。同步并行 $\parallel_A$ 允许对通信粒度进行精确控制——只有列入同步集 $A$ 的事件才需要多方同时就绪；其余事件可以自由交错。隐藏算子 $\setminus A$ 则支持模块化设计，将模块内部的协调细节转化为不可见的 $\tau$ 动作，从而在不同抽象层次上验证系统性质。

**定义动机**：如果不将通信事件限制为静态命名集合，就无法在编译期或模型检验前确定进程间的通信拓扑，从而失去穷尽验证的可行性。CSP 的静态命名设计使得工业级模型检验工具（如 FDR4）能够对有限状态子集进行完全验证 [^3]。此外，成功终止 $\text{SKIP}$ 与死锁 $\text{STOP}$ 的显式区分，使得 CSP 可以精细地建模进程的生命周期和精化关系（refinement），这是 traces/failures/divergences 语义域的根基。

### Def-S-05-02 (CSP 结构化操作语义)

CSP 的操作语义采用 Plotkin 风格的结构化操作语义（SOS），定义进程之间的标记转移关系 $P \xrightarrow{\alpha} P'$，其中 $\alpha \in \Sigma \cup \{\tau\}$。以下为核心 SOS 规则集：

```
                a → P ─a→ P                              [Prefix]

                P ─a→ P'
[Ext-L] ───────────────────────
                P □ Q ─a→ P'

                Q ─a→ Q'
[Ext-R] ───────────────────────
                P □ Q ─a→ Q'

                P ─τ→ P'
[Int-L] ───────────────────────
                P ⊓ Q ─τ→ P'

                Q ─τ→ Q'
[Int-R] ───────────────────────
                P ⊓ Q ─τ→ Q'

                P ─a→ P'
[Inter-L] ─────────────────────────
                P ||| Q ─a→ P' ||| Q

                Q ─a→ Q'
[Inter-R] ─────────────────────────
                P ||| Q ─a→ P ||| Q'

                P ─a→ P'         Q ─a→ Q'
[Sync] ───────────────────────────────────────
                P |[A]| Q ─a→ P' |[A]| Q'      (a ∈ A)

                P ─a→ P'         a ∉ A
[Sync-L] ───────────────────────────────────────
                P |[A]| Q ─a→ P' |[A]| Q

                Q ─a→ Q'         a ∉ A
[Sync-R] ───────────────────────────────────────
                P |[A]| Q ─a→ P |[A]| Q'

                P ─a→ P'         a ∈ A
[Hide] ───────────────────────────────────────
                P \ A ─τ→ P' \ A

                P ─a→ P'         a ∉ A
[Hide-Vis] ─────────────────────────────────────
                P \ A ─a→ P' \ A

                P ─a→ P'         a ≠ ✓
[Seq-L] ───────────────────────────────────────
                P ; Q ─a→ P' ; Q

                P ─✓→ P'         (P 成功终止)
[Seq-R] ───────────────────────────────────────
                P ; Q ─τ→ Q

                F[μX.F / X] ─a→ P'
[Rec] ───────────────────────────────────────
                μX.F ─a→ P'
```

**规则说明**：

- **[Prefix]** 是最基本的动作规则：前缀进程 $a \to P$ 执行事件 $a$ 后归约为 $P$。
- **[Ext-L]** 与 **[Ext-R]** 表明外部选择 $\square$ 的转移由子进程驱动：当某个子进程可以执行 $\alpha$ 时，整个外部选择也可以执行 $\alpha$ 并进入对应子进程的后续状态。这意味着外部选择的初始可执行事件集合是其各分支首事件的并集。
- **[Int-L]** 与 **[Int-R]** 表明内部选择 $\sqcap$ 可以自发地（通过内部 $\tau$ 动作）解决非确定性，转移到其中一个子进程。环境无法阻止这一内部决策。
- **[Inter-L]** 与 **[Inter-R]** 描述交错并行 $|||$：两个子进程独立执行，其动作以任意顺序交错出现。
- **[Sync]** 是 CSP 同步通信的核心：当事件 $a$ 属于同步集 $A$ 时，$P$ 和 $Q$ 必须**同时**准备好执行 $a$ 才能进行转移。这实现了多进程间的**握手式同步**。
- **[Sync-L]** 与 **[Sync-R]** 处理不在同步集 $A$ 中的事件：这些事件可以由任一方独立执行，另一方保持不变。这体现了同步并行的**局部独立性**。
- **[Hide]** 与 **[Hide-Vis]** 描述隐藏算子：$A$ 中的事件被转化为内部 $\tau$（不可观察），不在 $A$ 中的事件保持可见。
- **[Seq-L]** 与 **[Seq-R]** 描述顺序组合：在 $P$ 未完成前（未发出 $\checkmark$），$P; Q$ 跟随 $P$ 的转移；一旦 $P$ 成功终止（发出 $\checkmark$），$P; Q$ 通过内部 $\tau$ 转移到 $Q$。
- **[Rec]** 描述递归：递归进程 $\mu X.F$ 的行为等价于将 $F$ 中的 $X$ 替换为 $\mu X.F$ 自身后的行为。

**直观解释**：SOS 规则将抽象的 CSP 语法转换为可执行的标记转移系统（LTS）。每一条规则都对应一个特定的组合算子，说明当子进程发生一步转移时，父进程应如何响应。同步规则 **[Sync]** 是 CSP 区别于其他进程代数（如 CCS 的二元标签匹配）的关键：它支持**多方同步**（multi-party synchronization），即任意多个进程可以在同一事件上同时握手，只要它们都列入了该事件的同步集。

**定义动机**：如果没有 SOS，CSP 的语法只是无意义的符号排列。SOS 赋予了每个算子精确的动态含义，使得我们可以在此基础上定义迹语义、失败语义和互模拟等价。特别是 **[Sync]** 规则，它直接支撑了 CSP 在工程中的应用——例如 Go 的无缓冲 channel 通信、Occam 的并行构造，都可以被严格地归约为该规则下的行为实例 [^2][^3]。

### Def-S-05-03 (CSP 迹语义、失败语义与发散语义)

CSP 拥有三个层次递进的语义模型：迹语义（Trace Semantics）、失败语义（Failures Semantics）和失败-发散语义（Failures-Divergences Semantics）。这三个模型从粗到细地刻画了进程的可观察行为，分别对应不同的验证精度和可判定性 [^3]。

#### 迹语义 (Trace Semantics)

设 $\Sigma^*$ 为事件字母表上的有限序列集合，$\varepsilon$ 为空序列。进程 $P$ 的**迹集合** $\text{traces}(P) \subseteq (\Sigma \cup \{\checkmark\})^*$ 归纳定义如下：

$$
\begin{aligned}
\text{traces}(\text{STOP}) &= \{\varepsilon\} \\
\text{traces}(\text{SKIP}) &= \{\varepsilon, \langle \checkmark \rangle\} \\
\text{traces}(a \to P) &= \{\varepsilon\} \cup \{a \cdot s \mid s \in \text{traces}(P)\} \\
\text{traces}(P \mathbin{\square} Q) &= \text{traces}(P) \cup \text{traces}(Q) \\
\text{traces}(P \mathbin{\sqcap} Q) &= \text{traces}(P) \cup \text{traces}(Q) \\
\text{traces}(P \mathbin{|||} Q) &= \{\text{interleave}(s, t) \mid s \in \text{traces}(P), t \in \text{traces}(Q)\} \\
\text{traces}(P \mathbin{\parallel_A} Q) &= \{s \in (\Sigma \cup \{\checkmark\})^* \mid s \restriction A \in \text{traces}(P) \restriction A \cap \text{traces}(Q) \restriction A, \\ &\qquad\qquad\qquad\quad s \restriction (\Sigma \setminus A) \in \text{interleave}(\text{traces}(P), \text{traces}(Q))\} \\
\text{traces}(P \setminus A) &= \{s \restriction (\Sigma \setminus A) \mid s \in \text{traces}(P)\} \\
\text{traces}(P; Q) &= \text{traces}(P) \mathbin{;\!\!_\checkmark} \text{traces}(Q) \\
\text{traces}(\mu X.F) &= \bigcup_{n \geq 0} \text{traces}(F^n[\text{STOP}/X])
\end{aligned}
$$

其中：

- $s \restriction A$ 表示将序列 $s$ 限制在事件集 $A$ 上的投影（删除不在 $A$ 中的元素）。
- $\text{interleave}(s, t)$ 表示 $s$ 与 $t$ 的所有可能交错序列。
- $T_1 \mathbin{;\!\!_\checkmark} T_2$ 为带成功终止的连接运算：若 $s \in T_1$ 不含 $\checkmark$，则 $s \in T_1 \mathbin{;\!\!_\checkmark} T_2$；若 $s = s' \cdot \langle \checkmark \rangle$ 且 $t \in T_2$，则 $s' \cdot t \in T_1 \mathbin{;\!\!_\checkmark} T_2$。
- $F^n[\text{STOP}/X]$ 表示对 $F$ 进行 $n$ 次代入展开，从 $\text{STOP}$ 开始。

#### 失败语义 (Failures Semantics)

失败语义不仅记录进程“能做什么”（迹），还记录进程在某些迹之后“可能拒绝做什么”（refusals）。这对于区分外部选择和内部选择至关重要。进程 $P$ 的**失败集合** $\text{failures}(P)$ 定义为：

$$
\text{failures}(P) = \{(s, X) \mid s \in \text{traces}(P), X \subseteq \Sigma, P \text{ after } s \text{ can refuse } X\}
$$

其中 $P \text{ after } s$ 表示 $P$ 执行迹 $s$ 后可能到达的进程集合，$Q$ 可以拒绝 $X$（记作 $Q \text{ ref } X$）当且仅当对于所有 $a \in X \cup \{\tau\}$，$Q$ 不能执行 $a$。直观地说，$(s, X)$ 表示在观察到迹 $s$ 之后，进程可能处于某个状态，该状态拒绝与环境进行 $X$ 中的任何事件交互。

例如，考虑 $P = a \to \text{STOP} \mathbin{\square} b \to \text{STOP}$ 和 $Q = a \to \text{STOP} \mathbin{\sqcap} b \to \text{STOP}$。在迹 $\varepsilon$ 之后，$P$ 可以拒绝 $\{b\}$（当环境选择 $a$ 分支时，$b$ 分支不再可用），但 $P$ 不能拒绝 $\{a, b\}$（因为至少有一个分支可用）。然而 $Q$ 通过内部 $\tau$ 可能已经偷偷选择了 $a \to \text{STOP}$，因此 $Q$ 可以拒绝 $\{b\}$，甚至可能拒绝 $\{a, b\}$（如果内部选择后到达 STOP）。因此 $\text{failures}(P) \neq \text{failures}(Q)$，但 $\text{traces}(P) = \text{traces}(Q)$。这揭示了失败语义比迹语义更精细。

#### 发散语义 (Divergences Semantics)

**发散**（divergence）指进程陷入无限执行的内部 $\tau$ 循环，不再产生任何可见事件，也不再成功终止。进程 $P$ 的**发散集合** $\text{divergences}(P)$ 定义为：

$$
\text{divergences}(P) = \{s \cdot t \mid s \in \text{traces}(P), \exists Q \in (P \text{ after } s). Q \xrightarrow{\tau^\omega}\}
$$

其中 $Q \xrightarrow{\tau^\omega}$ 表示从 $Q$ 出发存在无限长的纯 $\tau$ 转移序列。发散语义对于分析系统的**活锁**（livelock）至关重要：一个进程可能永远不会死锁（即永远有内部动作可执行），但从外部观察者的角度看，它陷入了不可见的无限循环，无法完成任何有用工作。

隐藏算子 $\setminus A$ 是引入发散的主要构造：如果 $P$ 在事件集 $A$ 上无限循环通信（例如 $P = a \to P$ 且 $a \in A$），则 $P \setminus A$ 会无限执行内部 $\tau$ 而发散。这在 Go 等工程语言中没有直接对应物，构成了 CSP 与实现语言之间的重要语义边界 [^3]。

**直观解释**：迹语义回答“进程可能产生哪些可见事件序列”；失败语义进一步回答“在某一历史之后，进程可能拒绝哪些事件”；发散语义则回答“进程是否可能陷入不可见的无限循环”。这三个语义层次构成了 CSP 精化关系 $P \sqsubseteq Q$（$Q$ 是 $P$ 的实现）的数学基础：迹精化（$T$）要求实现的迹是规范的子集；失败精化（$F$）还要求拒绝集包含规范的拒绝集；失败-发散精化（$FD$）进一步要求实现不能引入规范中没有的发散。

**定义动机**：如果只使用迹语义，外部选择 $\square$ 和内部选择 $\sqcap$ 将无法区分，导致无法验证某些安全性属性（例如“进程必须允许环境选择某个分支”）。失败语义的引入正是为了捕捉这种由环境可控性与进程自主性带来的差异。发散语义的引入则是为了检测活锁——在工业并发系统中，活锁往往比死锁更隐蔽、更危险。Roscoe 在 *The Theory and Practice of Concurrency* 中系统阐述了这三个语义模型，并据此设计了 FDR4 模型检验器的核心算法 [^3]。

### Def-S-05-04 (CSP 通道与同步原语)

在 CSP 中，进程间通信通过**事件**（event）进行，而事件通常聚集在**通道**（channel）上。通道是事件名的结构化集合，支持输入、输出和同步三种视角。设 $\mathcal{C}$ 为通道名集合，$\mathcal{V}$ 为值域，则 CSP 的通道事件字母表可定义为：

$$
\Sigma = \bigcup_{c \in \mathcal{C}} \{c.v \mid v \in \mathcal{V}_c\} \cup \{\checkmark\}
$$

其中 $c.v$ 表示“在通道 $c$ 上传递值 $v$ 的同步事件”。CSP 的经典语法中前缀 $a \to P$ 的 $a$ 可以是任意事件；为了更清晰地表达通道通信，我们引入如下**语法糖**（syntactic sugar）：

- **输入前缀**：$c?x \to P$ 表示“在通道 $c$ 上接收值 $x$，然后执行 $P$”。其语义为：
  $$
  c?x \to P \equiv \square_{v \in \mathcal{V}_c} (c.v \to P[v/x])
  $$
  即，对于每一个可能的值 $v$，进程都准备一个可以执行 $c.v$ 并绑定 $x = v$ 的分支。在实际验证中，值域通常是有限的或类型化的。

- **输出前缀**：$c!v \to P$ 表示“在通道 $c$ 上发送值 $v$，然后执行 $P$”。其语义为：
  $$
  c!v \to P \equiv c.v \to P
  $$
  输出前缀是一个单一事件，发送方主动提供值 $v$。

- **同步事件**：$c.v$ 本身既是输入也是输出。在同步并行 $P \mathbin{\parallel_A} Q$ 中，若 $c.v \in A$，则 $P$ 和 $Q$ 必须同时准备好参与 $c.v$ 才能进行转移。这与 Go 的无缓冲 channel 通信完全一致：发送方和接收方必须同时就绪，通信才能发生 [^6][^8]。

**多路同步**（Multi-way Synchronization）是 CSP 通道模型的独特优势。与 CCS 的二元握手（$a$ 与 $\bar{a}$ 配对）或 π-演算的点对点通信不同，CSP 的同步并行支持任意多个进程在同一事件上同步。例如，若有三个进程 $P, Q, R$ 都希望在事件 $c.v$ 上同步，可以构造：

$$
S = (P \mathbin{\parallel_{\{c.v\}}} Q) \mathbin{\parallel_{\{c.v\}}} R
$$

当且仅当 $P, Q, R$ 都准备好执行 $c.v$ 时，整个组合 $S$ 才能执行 $c.v$。这种机制在工程上对应于**屏障同步**（barrier synchronization）或**广播确认**（broadcast acknowledgement），是 CSP 能够直接建模某些分布式协议的原因之一。

**通道方向性与 Alphabet 约束**：在 CSP 中，通道的方向性（单向发送、单向接收、双向）通过限制进程的 **Alphabet**（即进程可能参与的事件集合）来实现。例如：

- 若进程 $P$ 的 alphabet 仅包含 $c!v$ 类事件，则 $P$ 永远不会在 $c$ 上接收值。
- 若进程 $Q$ 的 alphabet 仅包含 $c?x$ 类事件，则 $Q$ 永远不会在 $c$ 上发送值。
这与 Go 的 `chan<- T`（仅发送）和 `<-chan T`（仅接收）类型约束完全对应 [^7][^8]。

**直观解释**：CSP 的通道不是消息队列，而是**同步握手点**。通信事件 $c.v$ 是原子性的、瞬时的，发送方和接收方（甚至多方）在同一逻辑时刻共同完成该事件。这种设计消除了异步消息传递中的缓冲、排队、FIFO 顺序等复杂性，使得语义模型更加简洁，也更容易进行模型检验。然而，当需要表达异步语义时（如 Go 的缓冲 channel），必须在 CSP 中引入额外的辅助进程（如 $Buffer(c, n)$）来显式模拟队列状态 [^3][^7]。

**定义动机**：如果不将通道通信建模为同步事件，就无法直接借用 CSP 的迹语义和失败语义来分析 Go 的 `chan` 操作。建立 $c.v$ 作为单一同步事件的抽象，是证明 Go-CS-sync 与 CSP 迹等价（Thm-S-05-01）的关键桥梁。同时，多路同步机制扩展了 CSP 在分布式系统建模中的应用范围，使得它不仅仅是一个“两点握手”的理论玩具，而是能够描述真实工程中的同步原语 [^2][^3]。

---

### Def-S-05-05 (Go-CS-sync 到 CSP 的编码函数)

为了将 Go 语言的并发子集严格纳入 CSP 的数学框架，我们定义编码函数 $[\![ \cdot ]\!] : \text{Go-CS-sync} \to \text{CSP}$。Go-CS-sync 是 Go 的并发核心子集，仅使用无缓冲 channel，去除了缓冲 channel、`panic/recover`、闭包捕获等超出经典 CSP 表达能力的特性 [^7]。

Go-CS-sync 的抽象语法为：

$$
\begin{aligned}
P, Q ::= &\ 0 \quad \text{(终止 / 空语句)} \\
       |\ &\ \text{go}\ P \quad \text{(创建 goroutine)} \\
       |\ &\ ch \leftarrow v \quad \text{(向 channel $ch$ 发送值 $v$)} \\
       |\ &\ x := \leftarrow ch \quad \text{(从 channel $ch$ 接收)} \\
       |\ &\ \text{select}\ \{\ \text{case}_i : P_i\ \} \quad \text{(外部选择)} \\
       |\ &\ P; Q \quad \text{(顺序组合)} \\
       |\ &\ \text{rec}\ X.P \quad \text{(递归)}
\end{aligned}
$$

编码函数 $[\![ \cdot ]\!]$ 定义如下：

| Go 构造 | CSP 编码 | 语义说明 |
|---------|----------|----------|
| $0$ | $\text{STOP}$ | 终止进程，无后续动作 |
| $\text{go}\ P$ | $[\![P]\!] \mathbin{|||} \text{SKIP}$ | goroutine 映射为独立的交错并行进程 |
| $ch \leftarrow v; P$ | $ch.v \to [\![P]\!]$ | 发送映射为同步事件输出前缀 |
| $x := \leftarrow ch; P$ | $ch?x \to [\![P]\!]$ | 接收映射为输入前缀 |
| $P; Q$ | $[\![P]\!]; [\![Q]\!]$ | 顺序组合保持 |
| $\text{select}\ \{\ \text{case}_i : P_i\ \}$ | $\square_i\ (a_i \to [\![P_i]\!])$ | `select` 映射为外部选择，$a_i$ 为对应分支的首事件 |
| $\text{rec}\ X.P$ | $\mu X.[\![P]\!]$ | 递归映射为 CSP 的 $\mu$ 算子 |

其中，对于 `select` 的编码，若第 $i$ 个分支是发送操作 $ch_i \leftarrow v_i$，则 $a_i = ch_i.v_i$；若第 $i$ 个分支是接收操作 $x_i := \leftarrow ch_i$，则 $a_i = ch_i?x_i$。所有分支的首事件共同构成外部选择的初始可执行事件集。

**直观解释**：该编码函数将 Go 的并发原语逐条翻译为 CSP 的对应算子。`go P` 被映射为交错并行 $|||$，因为新创建的 goroutine 与父 goroutine 独立调度，不共享隐式同步关系（除非通过 channel）。channel 的发送和接收被映射为同一个同步事件 $ch.v$ 的不同视角，这精确刻画了无缓冲 channel 的握手语义。`select` 被映射为外部选择 $\square$，因为 `select` 的 case 选择依赖于环境中哪个 channel 操作就绪，这与外部选择“由环境决定分支”的语义完全一致 [^6][^7]。

**定义动机**：如果不显式定义这一编码函数，"Go 的并发模型基于 CSP"就只是一个工程直觉，无法被严格证明。通过 $[\![ \cdot ]\!]$，我们可以将 Go-CS-sync 程序的分析**归约**（reduce）到成熟的 CSP 理论框架上，从而应用 FDR4 等工具验证死锁自由、活锁自由和精化关系 [^3][^8]。此外，该编码也清晰地揭示了 Go-CS-sync 的表达能力边界：任何无法被该编码覆盖的 Go 特性（如缓冲 channel、panic/recover、闭包共享状态）都标志着 Go 与 CSP 之间的语义鸿沟。

## 2. 属性推导 (Properties)

从上述定义出发，本节推导 CSP 及其与 Go 映射的关键局部性质。这些性质既是独立有趣的理论结果，也是后续形式证明的引理基础。

### Property 1 (CSP 前缀的迹前缀闭包性)

**陈述**：对于任意前缀进程 $a \to P$，其迹集合 $\text{traces}(a \to P)$ 是前缀闭包的（prefix-closed），即若 $s \in \text{traces}(a \to P)$ 且 $t$ 是 $s$ 的前缀，则 $t \in \text{traces}(a \to P)$。

**推导**：

1. 由 Def-S-05-03，$\text{traces}(a \to P) = \{\varepsilon\} \cup \{a \cdot s \mid s \in \text{traces}(P)\}$。
2. 对于任意 $s \in \text{traces}(P)$，序列 $a \cdot s$ 的所有前缀要么是 $\varepsilon$，要么是 $a \cdot s'$ 其中 $s'$ 是 $s$ 的前缀。
3. 由于 $\text{traces}(P)$ 本身也是前缀闭包的（由迹语义的归纳定义保证），$s' \in \text{traces}(P)$。
4. 因此 $a \cdot s' \in \text{traces}(a \to P)$，且 $\varepsilon \in \text{traces}(a \to P)$。
5. 得证：$\text{traces}(a \to P)$ 是前缀闭包的。 ∎

**工程意义**：前缀闭包性保证了 CSP 进程的历史行为永远不会被“撤销”。在模型检验中，这意味着如果某个迹 $s$ 是合法的，那么 $s$ 的任何前缀也都是合法的，从而允许使用前缀树（trie）等数据结构高效表示和搜索状态空间。

---

### Property 2 (CSP 交错并行的结合律与交换律)

**陈述**：交错并行 $|||$ 满足结合律和交换律，即：
$$
(P \mathbin{|||} Q) \mathbin{|||} R \;=\; P \mathbin{|||} (Q \mathbin{|||} R), \qquad P \mathbin{|||} Q \;=\; Q \mathbin{|||} P
$$

**推导**：

1. 由 Def-S-05-03，$\text{traces}(P \mathbin{|||} Q) = \{\text{interleave}(s, t) \mid s \in \text{traces}(P), t \in \text{traces}(Q)\}$。
2. 序列的交错运算满足结合律：$\text{interleave}(\text{interleave}(s, t), u) = \text{interleave}(s, \text{interleave}(t, u))$。这是因为无论先交错哪两个序列，最终都是将三个序列中的元素以任意满足内部顺序的方式混合。
3. 交错运算也满足交换律：$\text{interleave}(s, t) = \text{interleave}(t, s)$。因为交错只要求保持每个子序列内部的相对顺序，子序列之间的顺序没有优先级。
4. 因此，迹集合层面的结合律和交换律成立。由于 CSP 的迹语义完全决定了进程在这些性质上的行为，算子本身也满足结合律和交换律。
5. 得证。 ∎

**工程意义**：结合律与交换律意味着 goroutine 的启动顺序不影响最终的并发语义。在 Go 中，$(\text{go}\ P); (\text{go}\ Q); R$ 与 $(\text{go}\ Q); (\text{go}\ P); R$ 在可观察行为上是等价的，这对应于其 CSP 编码 $([\![P]\!] \mathbin{|||} \text{SKIP}); ([\![Q]\!] \mathbin{|||} \text{SKIP}); [\![R]\!]$ 的交换律实例。

---

### Property 3 (CSP 外部选择可执行分支集合单调性)

**陈述**：设 $\text{ready}(P)$ 为进程 $P$ 在初始状态可以立即执行的事件集合。对于外部选择，有：
$$
\text{ready}(P \mathbin{\square} Q) \;=\; \text{ready}(P) \cup \text{ready}(Q)
$$

**推导**：

1. 由 Def-S-05-02 的 SOS 规则 **[Ext-L]**，若 $P \xrightarrow{a} P'$，则 $P \mathbin{\square} Q \xrightarrow{a} P'$。因此 $\text{ready}(P) \subseteq \text{ready}(P \mathbin{\square} Q)$。
2. 同理，由 **[Ext-R]**，$\text{ready}(Q) \subseteq \text{ready}(P \mathbin{\square} Q)$。
3. 外部选择没有其他的初始转移规则：它不能自发产生内部 $\tau$ 动作来解决选择（那是内部选择 $\sqcap$ 的行为）。因此 $P \mathbin{\square} Q$ 的初始可执行事件只能来自 $P$ 或 $Q$。
4. 综上，$\text{ready}(P \mathbin{\square} Q) = \text{ready}(P) \cup \text{ready}(Q)$。
5. 得证。 ∎

**工程意义**：这一单调性保证了 Go 的 `select` 语句在映射为 CSP 外部选择后，其“哪些 case 可以立即执行”的判断与 Go 运行时的就绪检测完全一致（参见 Lemma-S-05-01）。这是验证 `select` 语义保持性的核心引理。

---

### Property 4 (CSP 同步并行在空同步集上的退化)

**陈述**：当同步集为空集时，同步并行退化为交错并行：
$$
P \mathbin{\parallel_\emptyset} Q \;=\; P \mathbin{|||} Q
$$

**推导**：

1. 由 Def-S-05-03，$\text{traces}(P \mathbin{\parallel_A} Q)$ 要求 $s \restriction A \in \text{traces}(P) \restriction A \cap \text{traces}(Q) \restriction A$ 且 $s \restriction (\Sigma \setminus A) \in \text{interleave}(\text{traces}(P), \text{traces}(Q))$。
2. 当 $A = \emptyset$ 时，$s \restriction \emptyset = \varepsilon$。因此第一个条件 trivially 成立（任何序列限制到空集都是空序列）。
3. 此时第二个条件变为 $s \restriction \Sigma \in \text{interleave}(\text{traces}(P), \text{traces}(Q))$，即 $s \in \text{interleave}(\text{traces}(P), \text{traces}(Q))$。
4. 这恰好等于 $\text{traces}(P \mathbin{|||} Q)$ 的定义。
5. 得证。 ∎

**工程意义**：该性质说明 CSP 的同步并行是交错并行的**泛化**。在 Go 中，两个不共享任何 channel 的 goroutine 的行为等价于交错执行，这对应于 $A = \emptyset$ 的情况；而当它们共享 channel 时，$A$ 包含该 channel 上的事件，同步并行强制握手发生。

---

### Property 5 (Go 无缓冲 channel 通信与 CSP 同步事件的原子性等价)

**陈述**：在 Go-CS-sync 中，无缓冲 channel 上的成功通信 $ch \leftarrow v$ 与对应的接收 $x := \leftarrow ch$ 对应于 Go 小步语义中的单一内部 $\tau$ 转移；在 CSP 编码中，这对应于单一同步事件 $ch.v$。

**推导**：

1. 由 Go 的操作语义（参见 [Go-Operational-Semantics](../../AcotorCSPWorkflow/deep/02-language-analysis/Go/Go-Operational-Semantics.md)），无缓冲 channel 的发送方在接收方就绪前阻塞。当发送方和接收方同时就绪时，它们通过一步内部转移共同完成通信，转移标签为 $\tau$ [^6][^7]。
2. 在全局观察视角下，这一同步动作被记录为可见事件 $ch!v$（发送方视角）或 $ch?v$（接收方视角），但在 CSP 的统一事件字母表中，它被抽象为单一事件 $ch.v$。
3. 由 Def-S-05-05，发送编码为 $ch.v \to [\![P]\!]$，接收编码为 $ch?x \to [\![Q]\!]$。在 CSP 的同步并行 $[\![P]\!] \mathbin{\parallel_{\{ch.v\}}} [\![Q]\!]$ 中，双方必须同时参与 $ch.v$ 才能执行该事件（规则 **[Sync]**）。
4. 因此，Go 的无缓冲通信与 CSP 的同步事件在“原子性”和“同时性”约束上等价：都需要发送方和接收方同时就绪，且对外部观察者表现为单一事件。
5. 得证。 ∎

**工程意义**：这一等价性是 Thm-S-05-01（迹语义等价）证明的**核心案例**。它说明了为什么 Go-CS-sync 可以完全嵌入 CSP 的 L3 层次，而缓冲 channel（异步通信）则需要引入额外的状态进程，跃迁至 L4 [^7]。

---

## 3. 关系建立 (Relations)

本节将 CSP 置于更广阔的进程代数谱系中，阐明其与 CCS、π-演算以及 Go 语言并发模型的关系。这些关系不仅有助于理解 CSP 的表达边界，也为后续证明 Go-CS-sync 与 CSP 的迹等价提供了理论定位。

### 关系 1: CSP 与 CCS 的语义不可比较性

CSP 与 CCS 虽然都使用静态通道和同步握手，但它们在语义域上存在根本差异。关于这两个演算的详细定义，参见 [01.02-process-calculus-primer.md](./01.02-process-calculus-primer.md) 中的 Def-S-02-01 与 Def-S-02-02。

**论证**：

- **语义域差异**：CSP 基于迹/失败/发散语义域，CCS 基于互模拟语义域。强互模拟严格细于迹等价（$\sim \Rightarrow =_T$），但与失败等价不可比：存在 $P \sim Q$ 但 $P \neq_F Q$（互模拟不保证环境拒绝集相同），也存在 $P =_F Q$ 但 $P \not\sim Q$（失败等价允许内部结构差异）。
- **终止观测**：CSP 区分死锁 $\text{STOP}$ 与成功终止 $\text{SKIP}$（后者产生特殊事件 $\checkmark$），而 CCS 的空进程 $0$ 同时扮演了终止和死锁的角色。这导致从 CSP 到 CCS 的编码必须额外模拟 $\checkmark$，但该模拟在 CCS 的弱互模拟下会与死锁混淆。
- **选择算子**：CSP 的外部选择 $\square$ 和内部选择 $\sqcap$ 在失败语义下可以被精确区分；CCS 的 $+$ 算子无法同时保留这两种选择在失败语义和互模拟语义下的双重区分。

因此，CSP 和 CCS 在**等价语义**维度上是不可比较的（记作 $\perp$）。它们可以互相模拟对方的计算行为，但无法保持对方的原始语义等价关系。

---

### 关系 2: CSP 与 π-演算的表达能力包含关系

π-演算是支持动态名字创建和传递（移动性）的进程代数，其详细定义参见 [01.02-process-calculus-primer.md](./01.02-process-calculus-primer.md) 中的 Def-S-02-03。

**论证**：

- **编码存在性（CSP → π）**：CSP 的同步通信可以通过 π-演算的点对点握手模拟。交错并行 $|||$ 对应 $\mid$，同步并行 $\parallel_A$ 可以通过共享通道上的握手实现，隐藏 $\setminus A$ 对应限制 $(\nu a)$。因此存在从 CSP 到 π-演算的迹保持编码。
- **分离结果（π ↛ CSP）**：π-演算支持运行时通过 $(\nu a)$ 创建新通道，并通过 $\bar{b}\langle a \rangle$ 将其传递给其他进程，从而动态改变通信拓扑。由 Def-S-05-04 的静态命名约束，CSP 无法在运行时生成新的事件名。Sangiorgi & Walker (2001) 严格证明了不存在保持同余性的从 π-演算到 CSP 的编码 [^4]。

因此，**CSP $\subset$ π-演算**（在计算行为表达能力上严格弱于）。这意味着 CSP 位于表达能力层次 **L₃**（进程代数层），而 π-演算位于 **L₄**（移动计算层）。

---

### 关系 3: Go-CS-sync 与 CSP 的迹语义等价关系

**论证**：

- **编码存在性**：由 Def-S-05-05，存在编码函数 $[\![ \cdot ]\!] : \text{Go-CS-sync} \to \text{CSP}$。对于正向编码，也可以对任意有限 CSP 进程 $P$ 构造对应的 Go-CS-sync 程序 $G(P)$：$\text{STOP} \mapsto 0$，$a \to P \mapsto ch \leftarrow v; G(P)$，$P \mathbin{\square} Q \mapsto \text{select}\ \{\text{case}: G(P), \text{case}: G(Q)\}$，$P \mathbin{|||} Q \mapsto \text{go}\ G(P); G(Q)$，$\mu X.F \mapsto \text{rec}\ X.G(F)$。
- **双向保持性**：Thm-S-05-01 将严格证明：对于任意 $Q \in \text{Go-CS-sync}$，有 $\text{traces}_{\text{Go}}(Q) = \text{traces}_{\text{CSP}}([\![Q]\!])$。反之，对于任意有限 CSP 进程 $P$，有 $\text{traces}_{\text{CSP}}(P) = \text{traces}_{\text{Go}}(G(P))$。

因此，**Go-CS-sync $\iff$ CSP**（在迹语义下表达能力等价）。这一定位说明：仅使用无缓冲 channel 的 Go 并发子集，其理论表达能力与经典 CSP 完全一致，均位于 L₃ 层次 [^7]。

---

### 关系 4: Go select 到 CSP 外部选择的语义映射

**论证**：

- **映射方向**：Go 的 `select { case_i: P_i }` 在语义上映射到 CSP 的外部选择 $\square$，而非内部选择 $\sqcap$。
- **关键依据**：`select` 的 case 选择依赖于环境中哪个 channel 操作就绪（外部驱动）。当多个 case 同时就绪时，Go 运行时通过伪随机算法选择一个执行；从外部观察者的角度看，这种选择是不可预测的，等价于非确定性。但这仍然是一种“环境就绪驱动的选择”，与 CSP 外部选择 $\square$ 的语义一致（参见 Property 3）。
- **边界条件**：若 `select` 包含 `default` 分支，则语义偏离外部选择，更接近 CSP 的滑垒选择或内部选择混合。这在反例 3 中将进一步讨论。

因此，关系类型为编码映射 $\mapsto$：`select { case_i: P_i }` $\mapsto$ $\square_i (a_i \to [\![P_i]\!])$。

---

## 4. 论证过程 (Argumentation)

本节建立两个核心引理，它们将作为证明 Thm-S-05-01 的关键中间步骤。

### Lemma-S-05-01 (CSP 外部选择可执行分支集合保持性)

**陈述**：设 $P = \square_{i \in I} (a_i \to P_i)$ 为 CSP 外部选择进程，其中 $I$ 为有限索引集。定义 $P$ 的初始可执行事件集为 $\text{ready}(P) = \{a_i \mid P \xrightarrow{a_i} P_i\}$。则：
$$
\text{ready}(P) = \bigcup_{i \in I} \{a_i\}
$$

**证明**：

1. **前提分析**：由 Def-S-05-02 的 SOS 规则 **[Ext-L]** 和 **[Ext-R]**，外部选择 $P \mathbin{\square} Q$ 的初始转移完全由其子进程的初始转移决定。
2. **构造/推导**：对于 $P = \square_{i \in I} (a_i \to P_i)$，每个分支 $a_i \to P_i$ 根据 **[Prefix]** 规则满足 $a_i \to P_i \xrightarrow{a_i} P_i$。将该规则与 **[Ext-L]** 或 **[Ext-R]** 反复应用，可得对于任意 $i \in I$，整体进程 $P$ 满足 $P \xrightarrow{a_i} P_i$。
3. **完备性**：外部选择没有内部 $\tau$ 规则来解决选择（那是内部选择 $\sqcap$ 的特性）。因此，$P$ 在初始状态不能执行任何不属于 $\{a_i\}$ 的事件，也不能执行 $\tau$。
4. **结论**：$\text{ready}(P) = \bigcup_{i \in I} \{a_i\}$。 ∎

---

### Lemma-S-05-02 (CSP 同步并行下迹前缀保持性)

**陈述**：对于任意 CSP 进程 $P, Q$ 和同步集 $A \subseteq \Sigma$，若 $P \mathbin{\parallel_A} Q \xrightarrow{\alpha} R$，则：

- $\text{traces}(R) \subseteq \text{traces}(P \mathbin{\parallel_A} Q)$；且
- 若 $\alpha$ 是可见事件（$\alpha \neq \tau$），则 $\alpha \cdot \text{traces}(R) \subseteq \text{traces}(P \mathbin{\parallel_A} Q)$。

**证明**：

1. **前提分析**：$P \mathbin{\parallel_A} Q$ 的一步转移只能由规则 **[Sync]**、**[Sync-L]** 或 **[Sync-R]** 产生（Def-S-05-02）。
2. **案例 1（同步转移，$\alpha \in A$）**：由 **[Sync]**，$P \xrightarrow{\alpha} P'$ 且 $Q \xrightarrow{\alpha} Q'$，且 $R = P' \mathbin{\parallel_A} Q'$。设 $s \in \text{traces}(R)$，则 $s \restriction A \in \text{traces}(P') \restriction A \cap \text{traces}(Q') \restriction A$ 且 $s \restriction (\Sigma \setminus A) \in \text{interleave}(\text{traces}(P'), \text{traces}(Q'))$。由于 $\text{traces}(P') \subseteq \text{traces}(P)$ 且 $\text{traces}(Q') \subseteq \text{traces}(Q)$（由迹语义的前缀闭包性），可知 $s \in \text{traces}(P \mathbin{\parallel_A} Q)$。进一步，对于可见事件 $\alpha$，序列 $\alpha \cdot s$ 限制到 $A$ 后为 $\alpha \cdot (s \restriction A)$，它同时属于 $\text{traces}(P)$ 和 $\text{traces}(Q)$（因为 $\alpha \cdot \text{traces}(P') \subseteq \text{traces}(P)$ 且同理对 $Q$）。因此 $\alpha \cdot s \in \text{traces}(P \mathbin{\parallel_A} Q)$。
3. **案例 2（独立转移，$\alpha \notin A$）**：不妨设由 **[Sync-L]**，$P \xrightarrow{\alpha} P'$ 且 $Q$ 不变，$R = P' \mathbin{\parallel_A} Q$。对于任意 $s \in \text{traces}(R)$，由于 $\text{traces}(P') \subseteq \text{traces}(P)$，同样的包含关系成立。若 $\alpha$ 可见，则 $\alpha \cdot s$ 限制到 $A$ 仍为 $s \restriction A$（因为 $\alpha \notin A$），该投影不变；而限制到 $\Sigma \setminus A$ 为 $\alpha \cdot (s \restriction (\Sigma \setminus A))$，它属于 $\text{interleave}(\text{traces}(P), \text{traces}(Q))$。因此 $\alpha \cdot s \in \text{traces}(P \mathbin{\parallel_A} Q)$。
4. **结论**：由案例分析，引理对所有可能的转移成立。 ∎

---

### 论证: 迹等价定理的关键步骤

Thm-S-05-01 旨在证明 Go-CS-sync 与 CSP 编码在迹语义下的等价性。其核心证明策略是**对 Go-CS-sync 进程 $Q$ 的结构进行归纳**（structural induction）：

1. **基础情形**：$Q = 0$（空进程）。其编码为 $\text{STOP}$，双方的迹集合均为 $\{\varepsilon\}$，等价性显然成立。
2. **前缀情形**：$Q$ 为发送 $ch \leftarrow v; P$ 或接收 $x := \leftarrow ch; P$。由 Property 5，Go 的同步通信对应 CSP 的同步事件 $ch.v$。结合归纳假设 $\text{traces}(P) = \text{traces}([\![P]\!])$，可直接推导出迹集合相等。
3. **选择情形**：$Q = \text{select}\ \{\text{case}_i : P_i\}$。由 Lemma-S-05-01，外部选择的初始可执行事件集恰好是其各分支首事件的并集，这与 Go `select` 的“就绪 case 集合”语义一致。结合归纳假设，迹集合相等。
4. **并行情形**：$Q = \text{go}\ P; R$。由 Property 2，交错并行满足结合律与交换律，这与 Go goroutine 的独立调度语义一致。结合 Lemma-S-05-02 保证同步并行下迹前缀的保持性，可证明整体迹集合相等。
5. **递归情形**：$Q = \text{rec}\ X.P$。由 Def-S-05-03，递归的迹语义定义为有限次展开的并集。编码将 $\text{rec}$ 映射为 $\mu$，两者的展开语义一致，因此等价性由归纳假设传递。

以下第 5 节将给出该定理的完整形式化证明。

## 5. 形式证明 (Proofs)

### Thm-S-05-01 (Go-CS-sync 与 CSP 编码保持迹语义等价)

**陈述**：对于任意 $Q \in \text{Go-CS-sync}$，有：
$$
\text{traces}_{\text{Go}}(Q) \;=\; \text{traces}_{\text{CSP}}([\![Q]\!])
$$
其中 $[\![ \cdot ]\!]$ 为 Def-S-05-05 定义的编码函数。

**证明**：

我们对 $Q$ 的语法结构进行归纳，证明对于所有子结构，迹语义保持性成立。

#### 基础情形 ($Q = 0$)

- 由 Go-CS-sync 语义，$0$ 为空语句，不执行任何可见动作。因此 $\text{traces}_{\text{Go}}(0) = \{\varepsilon\}$。
- 由 Def-S-05-05，$[\![0]\!] = \text{STOP}$。由 Def-S-05-03，$\text{traces}_{\text{CSP}}(\text{STOP}) = \{\varepsilon\}$。
- 因此 $\text{traces}_{\text{Go}}(0) = \text{traces}_{\text{CSP}}([\![0]\!])$。✓

#### 归纳步骤

假设对于所有严格小于 $Q$ 的子进程，迹语义保持性成立。以下对 $Q$ 的构造进行分类讨论。

---

**案例 1: 发送前缀 ($Q = ch \leftarrow v; P$)**

- 在 Go-CS-sync 中，无缓冲 channel 的发送操作必须与接收方同步完成。成功通信后，进程继续执行 $P$。Go 的迹语义将该同步动作记录为可见事件 $ch.v$（全局视角）。因此：
  $$
  \text{traces}_{\text{Go}}(ch \leftarrow v; P) = \{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{Go}}(P)\}
  $$
- 由 Def-S-05-05，编码为 $[\![ch \leftarrow v; P]\!] = ch.v \to [\![P]\!]$。
- 由 Def-S-05-03 的迹语义定义：
  $$
  \text{traces}_{\text{CSP}}(ch.v \to [\![P]\!]) = \{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P]\!])\}
  $$
- 由归纳假设，$\text{traces}_{\text{Go}}(P) = \text{traces}_{\text{CSP}}([\![P]\!])$。
- 因此 $\text{traces}_{\text{Go}}(ch \leftarrow v; P) = \text{traces}_{\text{CSP}}([\![ch \leftarrow v; P]\!])$。✓

**案例 2: 接收前缀 ($Q = x := \leftarrow ch; P$)**

- 与案例 1 对称。Go 的接收操作在同步完成后绑定变量 $x$ 为接收到的值 $v$，并继续执行 $P[x/v]$。在全局迹中，该动作同样记录为 $ch.v$。
- 编码为 $[\![x := \leftarrow ch; P]\!] = ch?x \to [\![P]\!]$。根据 Def-S-05-04 的语法糖，$ch?x \to [\![P]\!]$ 的迹语义为：
  $$
  \{\varepsilon\} \cup \{ch.v \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P]\!][v/x]), v \in \mathcal{V}_{ch}\}
  $$
- 由归纳假设，$\text{traces}_{\text{Go}}(P[x/v]) = \text{traces}_{\text{CSP}}([\![P]\!][v/x])$。
- 因此 $\text{traces}_{\text{Go}}(x := \leftarrow ch; P) = \text{traces}_{\text{CSP}}([\![x := \leftarrow ch; P]\!])$。✓

**案例 3: 外部选择 ($Q = \text{select}\ \{\text{case}_i : P_i\}_{i \in I}$)**

- 设第 $i$ 个分支的首事件为 $a_i$（若分支为发送则 $a_i = ch_i.v_i$，若为接收则 $a_i = ch_i?x_i$）。
- 在 Go 语义中，`select` 等待所有 case 中至少一个通信操作就绪。假设 case $i$ 就绪，则执行该通信后进入 $P_i$。因此：
  $$
  \text{traces}_{\text{Go}}(Q) = \{\varepsilon\} \cup \bigcup_{i \in I} \{a_i \cdot s \mid s \in \text{traces}_{\text{Go}}(P_i)\}
  $$
- 由 Def-S-05-05，编码为 $[\![Q]\!] = \square_{i \in I} (a_i \to [\![P_i]\!])$。
- 由 Def-S-05-03 的外部选择迹语义：
  $$
  \text{traces}_{\text{CSP}}(\square_{i \in I} (a_i \to [\![P_i]\!])) = \{\varepsilon\} \cup \bigcup_{i \in I} \{a_i \cdot s \mid s \in \text{traces}_{\text{CSP}}([\![P_i]\!])\}
  $$
- 由归纳假设，对每个 $i$，$\text{traces}_{\text{Go}}(P_i) = \text{traces}_{\text{CSP}}([\![P_i]\!])$。
- 因此 $\text{traces}_{\text{Go}}(Q) = \text{traces}_{\text{CSP}}([\![Q]\!])$。✓

**案例 4: 并行组合 ($Q = \text{go}\ P; R$)**

- 在 Go 语义中，`go P` 创建一个新的 goroutine 执行 $P$，当前 goroutine 继续执行 $R$。两者的执行迹以任意顺序交错，若共享 channel 则在对应事件上同步。
- 因此：
  $$
  \text{traces}_{\text{Go}}(\text{go}\ P; R) = \{s \mid s \in \text{interleave}(\text{traces}_{\text{Go}}(P), \text{traces}_{\text{Go}}(R)) \text{ with sync on shared channels}\}
  $$
- 由 Def-S-05-05，编码为 $[\![\text{go}\ P; R]\!] = ([\![P]\!] \mathbin{|||} \text{SKIP}); [\![R]\!]$。由于 $\text{SKIP}$ 只贡献 $\{\varepsilon, \langle \checkmark \rangle\}$，且 $[\![P]\!] \mathbin{|||} \text{SKIP}$ 的迹等价于 $[\![P]\!]$ 的迹（因为 $\text{SKIP}$ 不引入额外可见事件），整体编码的迹等价于 $[\![P]\!]$ 与 $[\![R]\!]$ 在共享 channel 事件集上的同步并行。
- 更精确地，设 $A$ 为 $P$ 和 $R$ 共享的 channel 事件集合。则 Go 的并发语义等价于 CSP 的 $[\![P]\!] \mathbin{\parallel_A} [\![R]\!]$。由 Def-S-05-03 的同步并行迹语义：
  $$
  \text{traces}_{\text{CSP}}([\![P]\!] \mathbin{\parallel_A} [\![R]\!]) = \{s \mid s \restriction A \in \text{traces}([\![P]\!]) \restriction A \cap \text{traces}([\![R]\!]) \restriction A,\ s \restriction (\Sigma \setminus A) \in \text{interleave}(\dots)\}
  $$
  这与 Go 的语义完全一致：共享 channel 上的事件必须双方同时参与（$s \restriction A$ 的交集约束），非共享事件自由交错（$\Sigma \setminus A$ 的交错约束）。
- 由归纳假设，$\text{traces}_{\text{Go}}(P) = \text{traces}_{\text{CSP}}([\![P]\!])$ 且 $\text{traces}_{\text{Go}}(R) = \text{traces}_{\text{CSP}}([\![R]\!])$。
- 因此 $\text{traces}_{\text{Go}}(\text{go}\ P; R) = \text{traces}_{\text{CSP}}([\![\text{go}\ P; R]\!])$。✓

**案例 5: 顺序组合 ($Q = P; R$)**

- 在 Go 语义中，$P; R$ 的迹是 $P$ 的迹与 $R$ 的迹的连接，但需去掉 $P$ 成功终止的标记（Go 中无显式的 $\checkmark$ 事件，成功终止即隐式进入 $R$）。
- 编码为 $[\![P; R]\!] = [\![P]\!]; [\![R]\!]$。由 Def-S-05-03 的顺序组合迹语义，$\text{traces}_{\text{CSP}}([\![P]\!]; [\![R]\!]) = \text{traces}_{\text{CSP}}([\![P]\!]) \mathbin{;\!\!_\checkmark} \text{traces}_{\text{CSP}}([\![R]\!])$。由于 Go 的迹中不含 $\checkmark$，该连接运算在限制到非 $\checkmark$ 事件后，恰好等于 Go 顺序组合的迹。
- 由归纳假设，$\text{traces}_{\text{Go}}(P) = \text{traces}_{\text{CSP}}([\![P]\!])$ 且 $\text{traces}_{\text{Go}}(R) = \text{traces}_{\text{CSP}}([\![R]\!])$。
- 因此 $\text{traces}_{\text{Go}}(P; R) = \text{traces}_{\text{CSP}}([\![P; R]\!])$。✓

**案例 6: 递归 ($Q = \text{rec}\ X.P$)**

- 在 Go 语义中，$\text{rec}\ X.P$ 等价于将 $P$ 中的 $X$ 替换为 $\text{rec}\ X.P$ 自身，即 $P[\text{rec}\ X.P / X]$。
- 因此：
  $$
  \text{traces}_{\text{Go}}(\text{rec}\ X.P) = \bigcup_{n \geq 0} \text{traces}_{\text{Go}}(P^{(n)})
  $$
  其中 $P^{(0)} = 0$，$P^{(n+1)} = P[P^{(n)} / X]$。
- 编码为 $[\![\text{rec}\ X.P]\!] = \mu X.[\![P]\!]$。由 Def-S-05-03：
  $$
  \text{traces}_{\text{CSP}}(\mu X.[\![P]\!]) = \bigcup_{n \geq 0} \text{traces}_{\text{CSP}}([\![P]\!]^{(n)})
  $$
  其中 $[\![P]\!]^{(0)} = \text{STOP}$，$[\![P]\!]^{(n+1)} = [\![P]\!][[\![P]\!]^{(n)} / X]$。
- 由归纳假设（结构归纳），对每个 $n$，$\text{traces}_{\text{Go}}(P^{(n)}) = \text{traces}_{\text{CSP}}([\![P]\!]^{(n)})$。
- 因此 $\text{traces}_{\text{Go}}(\text{rec}\ X.P) = \text{traces}_{\text{CSP}}([\![\text{rec}\ X.P]\!])$。✓

#### 结论

由结构归纳法，对于 Go-CS-sync 的所有语法构造，迹语义保持性成立。即对于任意 $Q \in \text{Go-CS-sync}$，
$$
\text{traces}_{\text{Go}}(Q) = \text{traces}_{\text{CSP}}([\![Q]\!])
$$
得证。 ∎

## 6. 实例验证 (Examples & Verification)

本节通过正例和反例验证上述形式化结果在工程实践中的适用性与边界。

### 示例 1: Pipeline 模式的形式映射

考虑以下 Go 程序，实现了一个从源产生整数、经过平方变换、最终由消费者接收的 Pipeline：

```go
func source(out chan int) {
    for i := 0; i < 3; i++ { out <- i }
    close(out)
}

func square(in, out chan int) {
    for x := range in { out <- x * x }
    close(out)
}

func main() {
    c1 := make(chan int)
    c2 := make(chan int)
    go source(c1)
    go square(c1, c2)
    for x := range c2 { fmt.Println(x) }
}
```

**CSP 编码**：

```csp
SOURCE = c1.0 → c1.1 → c1.2 → c1.end → STOP

SQUARE = c1?x → c2.(x*x) → SQUARE
         □
         c1.end → c2.end → STOP

MAIN = (SOURCE ||| SQUARE) ||| CONSUMER

CONSUMER = c2?x → print.x → CONSUMER
           □
           c2.end → STOP
```

**迹验证**：

1. `SOURCE` 产生的可见迹前缀为 $\langle c1.0, c1.1, c1.2, c1.end \rangle$。
2. `SQUARE` 在接收到 $c1.0$ 后输出 $c2.0$，接收到 $c1.1$ 后输出 $c2.1$，依此类推。其迹集合恰好是 `SOURCE` 迹经过变换后的像。
3. `CONSUMER` 的迹为 $\langle c2.0, print.0, c2.1, print.1, c2.4, print.4, c2.end \rangle$ 的所有前缀。
4. 整个系统的迹是上述三个进程在共享通道 $\{c1.v, c2.v, c1.end, c2.end\}$ 上同步后的结果。由 Thm-S-05-01，该 CSP 迹集合与原始 Go 程序的全局可观察事件序列完全一致。

---

### 示例 2: 外部选择与 Go select 的 Vending Machine

考虑经典的自动售货机 CSP 进程：

```csp
VM = coin → (tea → STOP □ coffee → STOP)
```

对应的 Go 实现：

```go
func vm(coin, tea, coffee chan struct{}) {
    <-coin
    select {
    case <-tea:
        // tea → STOP
    case <-coffee:
        // coffee → STOP
    }
}
```

**迹验证**：

1. CSP 进程 `VM` 的迹集合为 $\{\varepsilon, \langle coin \rangle, \langle coin, tea \rangle, \langle coin, coffee \rangle\}$。
2. Go 程序在执行 `<-coin` 后进入 `select`。若 `tea` 通道有信号，则选择第一个 case 并返回；若 `coffee` 有信号，则选择第二个 case。由于 `select` 在多个 case 同时就绪时伪随机选择，从外部观察者的角度看，迹集合同样为上述四组序列。
3. 由 Thm-S-05-01，该映射保持迹语义等价。此例也验证了关系 4（`select` 映射为外部选择 $\square$）的正确性。

---

### 反例 1: 缓冲 channel FIFO 语义在纯 CSP 迹等价中的丢失

**场景**：

```go
ch := make(chan int, 2)
go func() { ch <- 1; ch <- 2 }()
go func() { <-ch; <-ch }()
```

**分析**：

- Go 的缓冲 channel 保证 FIFO 顺序：若发送方按 $1, 2$ 发送，接收方必定按 $1, 2$ 接收。
- 然而，CSP 的经典迹语义只记录发生了哪些事件，不记录事件之间的因果顺序（除非使用更精细的偏序语义或带状态的缓冲进程）。
- 若尝试将缓冲 channel 编码为 CSP，需要引入辅助进程 $Buffer(ch, 2)$。但 $Buffer$ 进程内部存在额外的内部 $\tau$ 动作（值进入缓冲区、值离开缓冲区），这些 $\tau$ 动作使得迹集合与 Go 程序的直接迹等价被破坏。
- 更关键的是，纯 CSP 的迹语义无法区分“$1$ 先于 $2$ 被消费”的 FIFO 约束与“$1$ 和 $2$ 被任意顺序消费”的集合语义。因此，**带缓冲 channel 的 Go 程序无法通过简单的 CSP 编码保持 FIFO 顺序保证**。

**结论**：此反例揭示了 Go-CS-async（缓冲 channel）超出 CSP（L₃）表达能力的边界，其精确的 FIFO 语义需要引入更复杂的模型（如标记 Petri 网或显式状态缓冲进程），对应表达能力跃迁至 L₄ [^7]。

---

### 反例 2: CSP 隐藏算子的发散行为无法被 Go close(ch) 模拟

**CSP 进程**：

```csp
P = (a → P) \ {a}
```

在 CSP 中，$P$ 无限执行内部事件 $a$（被隐藏后变为 $\tau$），是一个**发散进程**（divergent process）。

**Go 尝试编码**：

```go
func P(a chan struct{}) {
    close(a)  // 只执行一次
}
```

**分析**：

- `close(a)` 只能执行一次，执行后 channel 进入关闭状态，后续接收立即返回零值。这不会产生无限循环的内部动作。
- 若尝试用 `for { a <- struct{}{} }` 模拟无限循环，则每次发送都是可见的同步事件（必须有接收方），无法被“隐藏”为不可见的内部 $\tau$。
- Go 语言没有任何原语能够将一个 channel 上的所有通信折叠为内部 $\tau$ 并允许无限循环而不被外部观察。

**结论**：`close(ch)` 不能完全模拟 CSP 的隐藏语义。CSP 的隐藏算子 $\setminus A$ 可以引入发散，这是 CSP 失败-发散语义的核心特征之一，而 Go 缺少直接的对应物 [^3][^7]。

---

### 反例 3: Go select with default 偏离 CSP 外部选择语义

**场景**：

```go
select {
case v := <-ch1:
    fmt.Println("ch1:", v)
case v := <-ch2:
    fmt.Println("ch2:", v)
default:
    fmt.Println("no data available")
}
```

**分析**：

- CSP 外部选择 $P \mathbin{\square} Q$ 的语义是：**等待**直到至少一个分支可执行，然后选择该分支。如果所有分支都阻塞，外部选择也阻塞。
- Go 的 `select` 带有 `default` 分支时，若所有通信 case 都未就绪，则**立即**执行 `default`，不会等待。
- 因此，`select with default` 在语义上更接近 CSP 的**滑垒选择**（sliding choice）或内部选择 $\sqcap$ 的混合，而非纯外部选择 $\square$。

**编码对比**：

```csp
-- CSP 外部选择（阻塞等待）
External = (ch1?x → P) □ (ch2?y → Q)

-- Go select with default（非阻塞）
GoSelect = (ch1?x → P) ⊓ (ch2?y → Q) ⊓ (default → R)
```

**结论**：在形式化分析中，必须显式区分无 `default` 的 `select`（映射为 $\square$）与带 `default` 的 `select`（映射为其他算子）。若混淆两者，可能导致对程序同步结构的错误推理 [^7]。
