## GitHub Actions 工作流设计

### 工作流分类

1. **持续集成 (CI)**: 代码提交时触发
2. **持续部署 (CD)**: 合并到主分支时触发
3. **定时任务**: 定时触发 (如每日构建)
4. **手动触发**: 人工触发 (如发布)

### 最佳实践

- 使用缓存加速构建
- 并行执行任务
- 设置超时时间
- 添加错误处理

### 安全检查

- 依赖漏洞扫描
- 代码安全扫描
- 密钥泄露检测

---

## 质量门禁

### 代码质量

- Lint检查通过
- 单元测试通过
- 代码覆盖率 > 80%

### 文档质量

- 链接检查通过
- Markdown格式正确
- 交叉引用有效
