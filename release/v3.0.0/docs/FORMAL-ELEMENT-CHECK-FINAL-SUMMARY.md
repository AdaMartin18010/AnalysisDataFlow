## 4. 编号连续性

### 4.1 序号空缺

发现 **379** 处序号空缺，示例：

- Thm-S-01: 序号空缺 1 → 3
- Thm-S-01: 序号空缺 5 → 9
- Thm-S-01: 序号空缺 9 → 30

### 4.2 分析

序号空缺**不一定需要修复**，可能原因：

1. **预留空间**: 为后续添加的元素预留编号
2. **历史删除**: 已删除的元素留下的空缺
3. **章节结构**: 不同章节使用不同编号范围

### 4.3 建议

- 核心文档（Struct/）建议保持连续性
- 可选文档（Knowledge/、Flink/）可接受适当空缺

## 5. 与注册表差异分析

### 5.1 差异原因

扫描结果（2,468）与 THEOREM-REGISTRY.md（9,357）存在显著差异，主要原因：

1. **规划 vs 实现**: 注册表包含规划中但未实际写入的元素

3. **统计口径**: 注册表可能包含不同层级的统计
4. **扫描局限**: 部分元素可能使用非标准格式未被识别
