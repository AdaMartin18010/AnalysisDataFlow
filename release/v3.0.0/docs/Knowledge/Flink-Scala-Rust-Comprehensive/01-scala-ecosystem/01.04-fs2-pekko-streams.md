## 1. 概念定义 (Definitions)

### Def-K-04-01: fs2 (Functional Streams for Scala)

**定义 (L4 形式化)**:

fs2 是一个纯函数式的流处理库，基于 Cats Effect 提供引用透明、可组合的流抽象。设 fs2 流类型为:

$$
\text{Stream}[F[_], A] \triangleq \{ s: \text{Pull}[F, A, \text{Unit}] \}
$$

其中:

- $F[_]$: 效应类型（通常为 `IO` 或 `Pure`）
- $A$: 元素类型
- $\text{Pull}$: 底层拉取计算

**核心操作语义**:

| 操作 | 类型签名 | 语义 |
|-----|---------|------|
| `map` | `(A => B) => Stream[F, A] => Stream[F, B]` | 函子映射 |
| `flatMap` | `(A => Stream[F, B]) => Stream[F, A] => Stream[F, B]` | 单子展开 |
| `filter` | `(A => Boolean) => Stream[F, A] => Stream[F, A]` | 过滤 |
| `fold` | `(B, (B, A) => B) => Stream[F, A] => Stream[F, B]` | 折叠 |
| `merge` | `Stream[F, A] => Stream[F, A] => Stream[F, A]` | 合并 |

**设计原则**:

1. **引用透明**: 流的定义不产生副作用
2. **惰性求值**: 元素按需产生
3. **资源安全**: 通过 `bracket` 保证资源释放
4. **可中断**: 支持取消和超时

---

### Def-K-04-02: Pekko Streams (Reactive Streams 实现)

**定义 (L4 形式化)**:

Pekko Streams 是 Apache Pekko 提供的 Reactive Streams 实现，基于图 (Graph) 的流处理抽象。设 Pekko Stream 为三元组:

$$
\text{Stream}_{Pekko} = \langle \text{Source}, \text{Flow}, \text{Sink} \rangle
$$

其中:

- **Source**: 流的起点，$\text{Source}[Out, Mat]$
- **Flow**: 流变换，$\text{Flow}[In, Out, Mat]$
- **Sink**: 流的终点，$\text{Sink}[In, Mat]$

**$Mat$ (物化值)**: 流运行后产生的辅助结果，如 `Future[Done]`。

**Reactive Streams 合规**:

Pekko Streams 完全实现 Reactive Streams 规范:

```
Publisher ──► Subscriber
     │            │
     ▼            ▼
  Source  ──►   Sink
     │            │
     └──► Flow ◄──┘
```

---

### Def-K-04-03: Pull-based vs Push-based 流模型

**定义 (L4 形式化)**:

**Pull-based 模型** (fs2):

$$
\text{Consumer} \xrightarrow{\text{request}} \text{Producer} \xrightarrow{\text{produce}} \text{Consumer}
$$

消费者主动拉取数据，生产者按需生成。

**Push-based 模型** (传统):

$$
\text{Producer} \xrightarrow{\text{push}} \text{Consumer}
$$

生产者主动推送数据，消费者被动接收。

**Reactive Streams 混合模型** (Pekko):

$$
\text{Consumer} \xrightarrow{\text{request(n)}} \text{Producer} \xrightarrow{\text{onNext(x)}} \text{Consumer}
$$

消费者订阅并请求数据量，生产者在约束下推送。

**模型对比**:

| 特性 | Pull-based (fs2) | Push-based | Reactive Streams |
|-----|-----------------|------------|------------------|
| 背压 | 自然支持 | 需额外处理 | 协议支持 |
| 延迟 | 按需生产 | 预生成 | 按需预生成 |
| 内存 | 低 (按需) | 可能高 (缓冲) | 可控 |
| 复杂度 | 高 | 低 | 中 |
| 适用场景 | 资源敏感 | 低延迟 | 通用 |

## 2. 属性推导 (Properties)

### Lemma-K-04-01: fs2 流的组合性

**引理**: fs2 流操作满足单子律，支持任意复杂度的组合。

**形式化表述**:

对于流 $s: \text{Stream}[F, A]$ 和操作 $f, g$:

**Functor 律**:
$$
\text{map}(\text{id}) = \text{id} \\
\text{map}(f \circ g) = \text{map}(f) \circ \text{map}(g)
$$

**Monad 律**:
$$
\text{flatMap}(\text{pure})(s) = s \\
\text{flatMap}(f)(\text{pure}(a)) = f(a) \\
\text{flatMap}(g)(\text{flatMap}(f)(s)) = \text{flatMap}(x \Rightarrow \text{flatMap}(g)(f(x)))(s)
$$

**证明**: 由 `Stream` 的 `Monad` 实例实现保证。

∎

**引理**: Pekko Streams 图结构支持运行时可视化，便于调试和监控。

### Prop-K-04-01: 背压传播保证

**命题**: fs2 和 Pekko Streams 都提供完整的背压传播机制，防止消费者过载。

**fs2 背压**:

```scala
// 消费者控制速率
stream.evalMap { item =>
  process(item)  // 处理完成后才拉取下一个
}

// 限速
stream.metered(100.millis)  // 每100ms最多一个元素
```

**Pekko Streams 背压**:

```scala
// 通过缓冲区控制
source.buffer(100, OverflowStrategy.backpressure)

// 限速
source.throttle(10, 1.second)
```

### 3.2 Reactive Streams 标准互操作

```java
// Publisher - 数据生产者
interface Publisher<T> {
    void subscribe(Subscriber<? super T> s);

interface Subscriber<T> {
    void onSubscribe(Subscription s);
    void onNext(T t);
    void onError(Throwable t);

interface Subscription {
```

## 4. 论证过程 (Argumentation)

### 4.1 fs2 纯函数式设计哲学

**核心原则**:

1. **引用透明**: 流的描述和执行分离

   ```scala
   // 这只是描述，没有执行
   val streamDesc: Stream[IO, Int] = Stream(1, 2, 3).map(_ * 2)

   // 这里才执行
   val result: IO[List[Int]] = streamDesc.compile.toList
   ```

2. **资源安全**: 通过 `bracket` 保证

   ```scala
   Stream.bracket(acquire)(use)(release)
   ```

3. **组合性**: 流可以任意组合而不改变语义

   ```scala
   val combined = stream1 ++ stream2  // 顺序组合
   val merged = stream1.merge(stream2)  // 并行合并
   ```

**与命令式流的对比**:

```scala
// 命令式 (回调风格)
source.onNext { item =>
  process(item)
  sink.send(transform(item))
}

// 函数式 (声明风格)
source.through(processPipe).through(transformPipe).to(sink)
```

### 4.2 Pekko Streams 的 Actor 模型集成

**Actor 与流的融合**:

```scala
// 使用 Actor 作为 Source
val actorSource: Source[String, ActorRef] =

// 使用 Actor 作为 Sink
val actorSink: Sink[String, NotUsed] =
  ActorSink.actorRef[String](targetActor, onCompleteMessage, onFailureMessage)
```

**Actor 流化**:

```scala
// Actor 处理流元素
class StreamProcessor extends Actor {

  }
}

// 将 Actor 包装为 Flow
val actorFlow: Flow[String, String, NotUsed] =
  Flow[String].ask[String](parallelism = 10)(actorRef)
```

## 5. 形式证明 / 工程论证 (Proof / Engineering Argument)

### Thm-K-04-01: fs2 流的引用透明性

**定理**: fs2 流满足引用透明性，即流的求值结果仅依赖于输入，不依赖外部状态。

**证明**:

**Step 1: 流的纯函数定义**

设 $S[F, A]$ 为 fs2 流，定义为:

$$
S[F, A] = \text{Free}[\text{Pull}[F, A, \_], \text{Unit}]
$$

**Step 2: 引用透明性验证**

对于任意 $s: S[F, A]$，$f: A \rightarrow B$:

$$
\text{eval}(s) = r \Rightarrow \forall s' = s. \text{eval}(s') = r
$$

**Step 3: 无副作用**

流的构造和组合不产生副作用，副作用仅在 `compile` 后执行:

```scala
// 纯函数 - 无副作用
val s1 = Stream.eval(IO.println("A"))
val s2 = Stream.eval(IO.println("A"))

// s1 和 s2 是等价的描述
```

∎

---

### 工程论证: 独立流库与 Flink 的互补关系

**论证**: fs2 和 Pekko Streams 与 Flink 不是竞争关系，而是互补关系。

**互补性分析**:

| 场景 | fs2/Pekko | Flink | 选择 |
|-----|----------|-------|------|
| 本地开发 | ✅ | ❌ | fs2/Pekko |
| 单元测试 | ✅ | ❌ | fs2/Pekko |
| 微服务 | ✅ | ⚠️ | fs2/Pekko |
| 批处理 (< 1GB) | ✅ | ⚠️ | fs2/Pekko |
| 流处理 (< 10K msg/s) | ✅ | ✅ | fs2/Pekko |
| 大规模分布式 | ❌ | ✅ | Flink |
| 状态管理 | ⚠️ | ✅ | Flink |
| Exactly-Once | ⚠️ | ✅ | Flink |

**推荐架构**:

```
开发阶段: fs2/Pekko (本地开发、测试)
     ↓
集成测试: fs2/Pekko + Kafka (集成验证)

```

## 6. 实例验证 (Examples)

### 6.1 fs2 核心操作详解

```scala
import fs2.{Stream, Pure, Pull}
import cats.effect.{IO, IOApp, Resource}
import cats.syntax.all._
import scala.concurrent.duration._

object Fs2CoreOperations extends IOApp.Simple {

  // ===== 纯数据流 =====
  val pureStream: Stream[Pure, Int] = Stream(1, 2, 3, 4, 5)

  // ===== 效应流 =====
  val ioStream: Stream[IO, String] = Stream.eval(IO {
    println("Computing...")
    "Result"
  })

  // ===== 变换操作 =====
  val transformed = pureStream
    .map(_ * 2)                    // [2, 4, 6, 8, 10]
    .filter(_ > 4)                 // [6, 8, 10]
    .take(2)                       // [6, 8]
    .drop(1)                       // [8]

  // ===== 扁平化 =====
  val nested: Stream[Pure, List[Int]] = Stream(
    List(1, 2),
    List(3, 4),
    List(5)
  )
  val flat: Stream[Pure, Int] = nested.flatMap(Stream.emits)
  // 结果: Stream(1, 2, 3, 4, 5)

  // ===== 折叠 =====
  val sum: Stream[Pure, Int] = pureStream.fold(0)(_ + _)
  // 结果: Stream(15) (单个元素)

  // ===== 扫描 (中间结果) =====
  val runningSum: Stream[Pure, Int] = pureStream.scan(0)(_ + _)
  // 结果: Stream(0, 1, 3, 6, 10, 15)

  // ===== 合并流 =====
  val stream1 = Stream(1, 2, 3)
  val stream2 = Stream(4, 5, 6)

  val concatenated: Stream[Pure, Int] = stream1 ++ stream2
  // 结果: Stream(1, 2, 3, 4, 5, 6)

  val interleaved: Stream[Pure, Int] = stream1.interleave(stream2)
  // 结果: Stream(1, 4, 2, 5, 3, 6)

  val merged: Stream[IO, Int] = stream1.covary[IO].merge(stream2.covary[IO])
  // 并发合并 (非确定性顺序)

  // ===== 资源管理 =====
  val resourceStream: Stream[IO, String] = Stream.bracket(
    IO { println("Acquiring resource"); scala.io.Source.fromFile("file.txt") }
  )(
    source => Stream.emits(source.getLines().toList)
  )(
    source => IO { println("Releasing resource"); source.close() }
  )

  // ===== 时间相关操作 =====
  val timedStream: Stream[IO, Int] = Stream
    .awakeEvery[IO](1.second)
    .zipRight(Stream(1, 2, 3, 4, 5))

  val debounced: Stream[IO, Int] = timedStream.debounce(500.millis)

  // ===== 背压控制 =====
  val metered: Stream[IO, Int] = pureStream
    .covary[IO]
    .metered(100.millis)  // 每 100ms 最多一个元素

  val buffered: Stream[IO, Int] = pureStream
    .covary[IO]
    .buffer(10)  // 缓冲区大小为 10

  // ===== 错误处理 =====
  val withErrorHandling: Stream[IO, Int] = Stream
    .eval(IO.raiseError(new Exception("Oops")))
    .handleErrorWith(e => Stream.eval(IO.println(s"Error: $e")) >> Stream.empty)

  // ===== 编译和运行 =====
  def run: IO[Unit] = {
    // 收集为 List
    val listIO: IO[List[Int]] = pureStream.compile.toList

    // 遍历每个元素
    val traverseIO: IO[Unit] = pureStream.evalMap(n => IO.println(n)).compile.drain

    // 折叠为单个值
    val foldIO: IO[Int] = pureStream.compile.fold(0)(_ + _)

    // 仅执行效应
    val drainIO: IO[Unit] = ioStream.compile.drain

    listIO.flatMap(list => IO.println(s"Result: $list"))
  }
}
```

### 6.2 Pekko Streams Graph DSL

```scala
import org.apache.pekko.actor.ActorSystem
import org.apache.pekko.stream.scaladsl._
import org.apache.pekko.stream.{ClosedShape, Graph, OverflowStrategy, UniformFanInShape, UniformFanOutShape}
import org.apache.pekko.{Done, NotUsed}
import scala.concurrent.{ExecutionContext, Future}

object PekkoStreamsGraphDSL {
  implicit val system: ActorSystem = ActorSystem("GraphDSLExample")
  implicit val ec: ExecutionContext = system.dispatcher

  // ===== 线性流 =====
  val source: Source[Int, NotUsed] = Source(1 to 100)
  val flow: Flow[Int, String, NotUsed] = Flow[Int].map(n => s"Number: $n")
  val sink: Sink[String, Future[Done]] = Sink.foreach(println)

  val linearGraph: RunnableGraph[Future[Done]] =
    source.via(flow).toMat(sink)(Keep.right)

  // ===== 扇出/扇入图 =====
  val fanOutFanInGraph: RunnableGraph[Future[Done]] =
    RunnableGraph.fromGraph(GraphDSL.create(sink) { implicit builder => out =>
      import GraphDSL.Implicits._

      val in = builder.add(Source(1 to 10))
      val broadcast = builder.add(Broadcast[Int](2))
      val merge = builder.add(Merge[String](2))

      val toEven = builder.add(Flow[Int].filter(_ % 2 == 0).map(n => s"Even: $n"))
      val toOdd = builder.add(Flow[Int].filter(_ % 2 == 1).map(n => s"Odd: $n"))

      in ~> broadcast ~> toEven ~> merge ~> out
            broadcast ~> toOdd  ~> merge

      ClosedShape
    })

  // ===== 平衡负载图 =====
  val balanceGraph: RunnableGraph[Future[Done]] =
    RunnableGraph.fromGraph(GraphDSL.create(Sink.ignore) { implicit builder => out =>
      import GraphDSL.Implicits._

      val in = builder.add(Source(1 to 1000))
      val balance = builder.add(Balance[Int](4))
      val merge = builder.add(Merge[Int](4))

      val worker = builder.add(Flow[Int].map(n => n * n).async)

      in ~> balance

      (0 until 4).foreach { i =>
        balance.out(i) ~> worker ~> merge.in(i)
      }

      merge ~> out

      ClosedShape
    })

  // ===== 带反馈的图 =====
  def feedbackGraph(threshold: Int): RunnableGraph[Future[Done]] = {
    RunnableGraph.fromGraph(GraphDSL.create(Sink.foreach[Int](println)) { implicit builder => out =>
      import GraphDSL.Implicits._

      val source = builder.add(Source(List(1, 2, 3)))
      val merge = builder.add(Merge[Int](2))
      val broadcast = builder.add(Broadcast[Int](2))

      val process = builder.add(Flow[Int].map(_ * 2))
      val filter = builder.add(Flow[Int].filter(_ < threshold))

      source ~> merge ~> process ~> broadcast ~> out
                         broadcast ~> filter ~> merge

      ClosedShape
    })
  }

  // ===== 分区图 =====
  val partitionGraph: RunnableGraph[Future[Done]] =
    RunnableGraph.fromGraph(GraphDSL.create(Sink.ignore) { implicit builder => out =>
      import GraphDSL.Implicits._

      val in = builder.add(Source(1 to 100))
      val partition = builder.add(Partition[Int](3, x => x % 3))

      val sink0 = builder.add(Sink.foreach[Int](x => println(s"Mod 0: $x")))
      val sink1 = builder.add(Sink.foreach[Int](x => println(s"Mod 1: $x")))
      val sink2 = builder.add(Sink.foreach[Int](x => println(s"Mod 2: $x")))

      in ~> partition
      partition.out(0) ~> sink0
      partition.out(1) ~> sink1
      partition.out(2) ~> sink2

      ClosedShape
    })

  // ===== 自定义图组件 =====
  def priorityMergeGraph[T](): Graph[UniformFanInShape[T, T], NotUsed] = {
    GraphDSL.create() { implicit builder =>
      import GraphDSL.Implicits._

      val merge = builder.add(MergePreferred[T](1))

      UniformFanInShape(merge.out, merge.in(0), merge.preferred)
    }
  }

  // ===== 带物化值的图 =====
  val graphWithMat: RunnableGraph[(Future[Done], Future[Int])] =
    source.toMat(Sink.fold(0)(_ + _))(Keep.both)

  def run(): Future[Done] = linearGraph.run()
}
```

```scala
// ===== fs2 背压 =====
object Fs2Backpressure {
  import fs2.Stream
  import cats.effect.IO
  import scala.concurrent.duration._

  // 自然背压 - 消费者控制速率
  def naturalBackpressure(producer: Stream[IO, Int]): Stream[IO, Int] =
    producer.evalMap { item =>
      // 模拟慢消费者
      IO.sleep(100.millis).as(item)
    }

  // 显式限速
  def throttledStream(source: Stream[IO, Int]): Stream[IO, Int] =
    source.metered(50.millis)  // 每秒最多 20 个

  // 带缓冲的背压
  def bufferedStream(source: Stream[IO, Int]): Stream[IO, Int] =
    source.buffer(100)  // 缓冲区大小 100

  // 自适应背压
  def adaptiveBackpressure(source: Stream[IO, Int]): Stream[IO, Int] =
    source.through(_.evalMap(IO.pure))  // 立即处理，自然背压
}

object PekkoBackpressure {
  import org.apache.pekko.stream.scaladsl._
  import org.apache.pekko.stream.OverflowStrategy
  import org.apache.pekko.NotUsed

  // 缓冲区背压
  def bufferedSource(source: Source[Int, NotUsed]): Source[Int, NotUsed] =
    source.buffer(100, OverflowStrategy.backpressure)

  // 限速
  def throttledSource(source: Source[Int, NotUsed]): Source[Int, NotUsed] =
    source.throttle(100, 1.second)

  // 采样
  def conflatedSource(source: Source[Int, NotUsed]): Source[Int, NotUsed] =

  // 扩展策略

    source.expand(i => Iterator.from(i))  // 重复最后一个元素
}
```

```scala
    properties.setProperty("bootstrap.servers", "localhost:9092")

      properties
    )

    val flinkStream: DataStream[String] = env.addSource(kafkaSource)

  // ===== Pekko Streams → Flink (通过 Reactive Streams) =====
  def pekkoToFlinkReactiveStreams(env: StreamExecutionEnvironment): Unit = {
    import org.reactivestreams.FlowAdapters

    // Pekko Source
    val pekkoSource: Source[String, NotUsed] = Source(1 to 100).map(_.toString)

    // 转换为 Reactive Streams Publisher
    val publisher = pekkoSource.toMat(Sink.asPublisher(false))(Keep.right).run()

    // 适配为 Flink Source
    // 注意: 需要自定义 ReactiveStreamsSourceFunction
    class ReactiveStreamsSourceFunction[T](
      publisher: org.reactivestreams.Publisher[T]
    )(implicit typeInfo: TypeInformation[T])
      extends RichSourceFunction[T] {

      @transient private var subscriber: ReactiveStreamsSubscriber[T] = _

      override def open(parameters: Configuration): Unit = {
        subscriber = new ReactiveStreamsSubscriber[T]()
        publisher.subscribe(subscriber)
      }

      override def run(ctx: SourceFunction.SourceContext[T]): Unit = {
        while (isRunning && !subscriber.isComplete) {
          subscriber.poll() match {
            case Some(element) => ctx.collect(element)
            case None => Thread.sleep(10)
          }
        }
      }

      override def cancel(): Unit = {
        subscriber.cancel()
      }
    }

    // 使用适配的 Source
    implicit val stringTypeInfo: TypeInformation[String] = TypeInformation.of(classOf[String])
    val adaptedSource = new ReactiveStreamsSourceFunction(
      FlowAdapters.toFlowPublisher(publisher)
    )

    val flinkStream: DataStream[String] = env.addSource(adaptedSource)
  }

  // ===== 共享文件系统集成 =====
  def fileSystemBridge(env: StreamExecutionEnvironment): Unit = {
    // fs2 写入文件
    val writeToFile = fs2.io.file.Files[IO].writeAll(
      java.nio.file.Paths.get("/tmp/output.parquet")

    val fs2Stream: Stream[IO, Byte] = ???
    val writeIO = fs2Stream.through(writeToFile).compile.drain

    // Flink 读取文件
    val flinkStream = env.readFile(
      new ParquetInputFormat(),

// Reactive Streams Subscriber 实现
class ReactiveStreamsSubscriber[T] extends org.reactivestreams.Subscriber[T] {
  @volatile private var subscription: org.reactivestreams.Subscription = _
  private val queue = new java.util.concurrent.ConcurrentLinkedQueue[T]()
  @volatile private var complete = false

  override def onSubscribe(s: org.reactivestreams.Subscription): Unit = {
    subscription = s
    s.request(1)
  }

  override def onNext(t: T): Unit = {
    queue.offer(t)
    subscription.request(1)
  }

  override def onError(t: Throwable): Unit = {
    complete = true
  }

  override def onComplete(): Unit = {
    complete = true
  }

  def poll(): Option[T] = Option(queue.poll())
  def isComplete: Boolean = complete && queue.isEmpty
  def cancel(): Unit = if (subscription != null) subscription.cancel()
}
```
