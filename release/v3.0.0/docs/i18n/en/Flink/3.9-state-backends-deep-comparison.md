<!-- TRANSLATE: ### Lemma-F-3.9-01: HashMapStateBackend 访问延迟上界 -->

<!-- TRANSLATE: **引理**: HashMapStateBackend 的状态访问延迟满足： -->

```
Latency_access < 100ns (99th percentile)
```

<!-- TRANSLATE: **证明概要**: -->

<!-- TRANSLATE: - 状态以 Java 对象形式存在于堆内存 -->
<!-- TRANSLATE: - HashMap 查找为 O(1) 操作 -->
<!-- TRANSLATE: - 无需序列化/反序列化开销 -->
<!-- TRANSLATE: - GC 停顿是主要变量因素 -->

<!-- TRANSLATE: ### Lemma-F-3.9-02: RocksDB 状态访问延迟特性 -->

<!-- TRANSLATE: **引理**: EmbeddedRocksDBStateBackend 的状态访问延迟满足： -->

```
Latency_access = L_disk + L_seek + L_deserialize

其中:
- L_disk: 磁盘 I/O 延迟 (SSD: 10-100μs, HDD: 1-10ms)
- L_seek: LSM-Tree 层数 × 每层查找 (通常 3-7 层)
- L_deserialize: 反序列化开销 (与对象复杂度相关)
```

<!-- TRANSLATE: ### Lemma-F-3.9-03: 内存状态容量约束 -->

<!-- TRANSLATE: **引理**: HashMapStateBackend 的最大状态容量受限于： -->

```
MaxStateSize = TM_Heap_Size × 0.7 - Overhead

Overhead 包含:
- Network Buffers (默认 10%)
- Managed Memory (默认 40%)
- JVM 元数据和开销 (约 20%)
```

<!-- TRANSLATE: ### Prop-F-3.9-01: Checkpoint 时间与状态大小关系 -->

<!-- TRANSLATE: **命题**: 对于完整 Checkpoint，完成时间与状态大小呈线性关系： -->

```
T_checkpoint = T_sync + T_async

T_sync (HashMap) = O(|State|)  // 序列化到内存缓冲区
T_sync (RocksDB) = O(1)        // 创建 SST 快照点

T_async = O(|State|) / Bandwidth_to_Storage
```

<!-- TRANSLATE: ### 增量 Checkpoint 原理 -->

<!-- TRANSLATE: **增量 Checkpoint 算法**: -->

```
定义:
- S_n: 第 n 次 Checkpoint 的状态集合
- ΔS_n: 第 n 次 Checkpoint 的增量变化

完整 Checkpoint: CP_n = S_n
增量 Checkpoint: CP_n = CP_{n-1} + ΔS_n

增量识别机制 (RocksDB):
1. 维护 SST 文件版本列表: {f_1, f_2, ..., f_k}
2. 新 SST 文件标记为 "NEW"
3. 已删除范围标记为 "DELETED"
4. Checkpoint 元数据: {base_checkpoint_id, new_files[], deleted_ranges[]}
```

<!-- TRANSLATE: **代码示例**: -->

```java
// 低延迟状态访问示例
public class LowLatencyRiskCheck extends KeyedProcessFunction<String, Transaction, Alert> {

    private ValueState<RuleCache> ruleState;
    private ValueState<UserSession> sessionState;

    @Override
    public void open(Configuration parameters) {
        // HashMapStateBackend 下状态直接以对象形式存储
        StateTtlConfig ttlConfig = StateTtlConfig
            .newBuilder(Time.minutes(5))
            .setUpdateType(OnCreateAndWrite)
            .setStateVisibility(NeverReturnExpired)
            .build();

        ValueStateDescriptor<RuleCache> ruleDescriptor =
            new ValueStateDescriptor<>("rules", RuleCache.class);
        ruleDescriptor.enableTimeToLive(ttlConfig);
        ruleState = getRuntimeContext().getState(ruleDescriptor);
    }

    @Override
    public void processElement(Transaction txn, Context ctx, Collector<Alert> out)
            throws Exception {
        // O(1) 内存访问，延迟 < 1μs
        RuleCache rules = ruleState.value();
        UserSession session = sessionState.value();

        // 风控规则检查
        RiskScore score = evaluateRisk(txn, rules, session);
        if (score.isHighRisk()) {
            out.collect(new Alert(txn.getId(), score));
        }

        // 更新会话状态
        session.update(txn);
        sessionState.update(session);
    }
}
```

<!-- TRANSLATE: **压缩策略选择**: -->

<!-- TRANSLATE: | 工作负载 | 推荐压缩策略 | 说明 | -->
