## 第四部分：编写形式化定义 (5 分钟)

### 画面

- VS Code 编辑界面
- 定义编写示例

### 脚本

```
现在开始编写形式化定义。

一个好的定义包含三个部分：
形式化定义、直观解释、和编号标注。

让我演示如何编写一个 Watermark 延迟边界的定义。

首先，编写形式化部分：

**定义 3.2** (Watermark 延迟边界): `Def-F-03-09`

形式化定义：
设事件流 $S$ 的事件时间戳为 $t(e)$，
Watermark 生成函数为 $w(t)$，
则延迟边界 $\delta$ 定义为：

$$
\delta = \max_{e \in S} \{ t_{proc}(e) - w(t(e)) \}
$$

其中 $t_{proc}(e)$ 是事件 $e$ 的处理时间。

然后，添加直观解释：

直观解释：
延迟边界 $\delta$ 表示事件实际处理时间与 Watermark 指示时间之间的最大差距。
它衡量了系统处理乱序事件所需的缓冲时间。

在实际应用中，延迟边界帮助我们：
1. 评估 Watermark 策略的有效性
2. 确定窗口计算的等待时间
3. 优化系统资源分配

最后，添加示例说明：

**示例**：
假设 Watermark 延迟设置为 5 秒，
某事件的事件时间为 10:00:00，
Watermark 时间为 10:00:05，
则该事件的延迟为 5 秒。
```

### 屏幕操作

```
1. 创建新文件或打开现有文件
2. 编写定义标题和编号
3. 编写形式化定义（使用 LaTeX）
4. 添加直观解释
5. 添加示例
```

## 第五部分：编写引理和性质 (3 分钟)

### 画面

- 引理编写示例
- 证明结构展示

### 脚本

```
接下来，我们学习如何编写引理和性质。

引理是从定义直接推导的性质，
通常作为定理证明的辅助。

让我编写一个关于延迟边界单调性的引理。

**引理 3.3** (延迟边界单调性): `Lemma-F-03-04`

**陈述**：
设 $\delta_1$ 和 $\delta_2$ 是两个不同的延迟边界，
如果 Watermark 生成策略保持不变，
则 $\delta_1 \leq \delta_2$ 当且仅当最大乱序度 $o_1 \leq o_2$。

**证明**：

1. 根据延迟边界定义：
   $$\delta = \max_{e \in S} \{ t_{proc}(e) - w(t(e)) \}$$

2. Watermark 生成函数 $w(t)$ 与最大乱序度 $o$ 相关：
   $$w(t) = t - o$$

3. 代入得：
   $$\delta = \max_{e \in S} \{ t_{proc}(e) - (t(e) - o) \}$$
   $$= \max_{e \in S} \{ t_{proc}(e) - t(e) \} + o$$

4. 因此，$\delta$ 与 $o$ 成正比，
   即 $\delta_1 \leq \delta_2 \Leftrightarrow o_1 \leq o_2$。

∎ **证毕**

注意证明的结构：
- 清晰的步骤编号
- 每步说明推理依据
- 最后使用 ∎ 标记证明结束
```

### 屏幕操作

```
1. 编写引理标题和编号
2. 编写陈述部分
3. 编写证明步骤
4. 添加证明结束标记
```

## 第六部分：编写主要定理 (5 分钟)

### 画面

- 定理编写示例
- 完整证明展示

### 脚本

```
最后，让我们学习如何编写主要定理。

定理是文档的核心成果，
需要完整的陈述和严格的证明。

让我编写一个关于延迟边界保证的定理。

**定理 3.4** (延迟边界保证): `Thm-F-03-06`

**陈述**：
给定事件流 $S$ 和 Watermark 生成策略 $W$，
设延迟边界为 $\delta$，
则对于任意窗口计算，
迟到数据的比例不超过 $P_{late}$，其中：

$$
P_{late} \leq \frac{|\{e \in S: delay(e) > \delta\}|}{|S|}
$$

**证明**：

**前提**：
- 事件流 $S = \{e_1, e_2, ..., e_n\}$
- 每个事件 $e_i$ 有事件时间 $t_i$ 和处理时间 $t'_i$
- Watermark 函数 $w(t) = t - \delta$

**步骤 1：定义迟到事件**

事件 $e$ 被认为是迟到的，当且仅当：
$$t(e) < w(t_{proc}(e))$$

即事件时间小于触发计算时的 Watermark 时间。

**步骤 2：推导迟到条件**

将 Watermark 函数代入：
$$t(e) < t_{proc}(e) - \delta$$
$$\Rightarrow t_{proc}(e) - t(e) > \delta$$
$$\Rightarrow delay(e) > \delta$$

**步骤 3：计算迟到比例**

迟到事件集合：
$$L = \{e \in S: delay(e) > \delta\}$$

迟到比例：
$$P_{late} = \frac{|L|}{|S|} = \frac{|\{e \in S: delay(e) > \delta\}|}{|S|}$$

**步骤 4：结论**

因此，迟到数据的比例恰好等于延迟超过边界的事件比例，
即：
$$P_{late} = \frac{|\{e \in S: delay(e) > \delta\}|}{|S|}$$

∎ **证毕**

**推论**：
如果事件延迟服从分布 $D$，
则可以通过调整 $\delta$ 来控制迟到率：
$$P_{late} = 1 - CDF_D(\delta)$$

其中 $CDF_D$ 是分布 $D$ 的累积分布函数。

这个定理为生产环境中的 Watermark 调优提供了理论依据。
```
