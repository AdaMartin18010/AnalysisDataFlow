# 性能数据引用与标注规范

> **版本**: v1.0 | **生效日期**: 2026-04-19 | **适用范围**: 全项目案例研究与基准测试文档

---

## 1. 为什么需要可信度标注

流计算领域的性能数据高度依赖测试环境。RisingWave 2026 权威对比报告指出：

- 相同 Nexmark 查询在不同硬件下差异可达 3x
- 状态规模直接影响恢复时间（Flink 1.x 分钟级 vs ForSt 秒级）
- 云成本对比必须标注区域和实例类型

**未标注来源的性能数据 = 不可信数据**

---

## 2. 可信度分级体系

所有性能数据必须标注以下三种标识之一：

### 📊 实测数据

```markdown
📊 **实测数据** | 环境: AWS r6g.2xlarge × 3, Flink 2.2.0, 100GB RocksDB 状态
吞吐量: 1.2M events/sec | 延迟: P99 45ms
```

**要求**：

- 必须附测试环境模板（见第 4 节）
- 必须说明测试工具（Nexmark / 自定义 / 其他）
- 必须说明测试时长和重复次数

### 📖 官方数据

```markdown
📖 **官方数据** | 来源: Apache Flink 官方博客, "Flink 2.2.0 Release", 2025-12-04
吞吐量: 1.5M events/sec (官方声明)
```

**要求**：

- 必须提供可验证的 URL 或文献引用
- 优先使用官方白皮书/博客/Release Notes
- 引用格式：`[^n]` 上标

### 🔮 估算数据

```markdown
🔮 **估算数据** | 置信区间: [0.8M, 1.5M] events/sec | 依据: 基于 Flink 1.18 数据线性外推 + 存算分离架构理论增益
```

**要求**：

- 必须说明估算方法和依据
- 必须提供置信区间或误差范围
- 禁止在选型决策树中作为唯一依据使用
- 必须在表格底部统一声明

---

## 3. 现有文档批量标注规范

对于历史文档中已有的性能数据：

1. **有明确来源的** → 标注为 `📖 官方数据` 或 `📊 实测数据`
2. **无来源但合理的行业共识** → 标注为 `🔮 估算数据`，补充说明依据
3. **明显过时或无法验证的** → 删除或替换为 `[数据待更新]`

---

## 4. 测试环境标准模板

所有实测数据必须包含以下信息（至少）：

```markdown
### 测试环境

| 维度 | 配置 |
|------|------|
| **硬件** | AWS r6g.2xlarge × 3 (8 vCPU, 64GB RAM) |
| **存储** | EBS gp3 3000 IOPS / S3 (ForSt 模式) |
| **网络** | 10Gbps 内网 |
| **Flink 版本** | 2.2.0 |
| **JDK 版本** | 17 (Temurin) |
| **状态后端** | RocksDB / ForSt / HashMap |
| **Checkpoint 间隔** | 30s |
| **数据集** | Nexmark Query 1-12 / 自定义 10亿条事件 |
| **测试时长** | 30 分钟稳定期 + 10 分钟测量期 |
| **重复次数** | 3 次取中位数 |
```

---

## 5. 竞品对比特殊规范

进行 Flink vs RisingWave / Materialize / Spark 等对比时：

1. **使用相同测试环境**（硬件、数据集、查询语义等价）
2. **标注版本号**（如 RisingWave v2.0, Flink v1.20）
3. **标注测试工具**（Nexmark 版本、查询子集）
4. **引用双方官方数据时，必须分别标注来源，不得混合**
5. **禁止选择性展示数据**（如只展示对一方有利的查询子集）

---

## 6. 审核清单

提交含性能数据的文档前，确认：

- [ ] 所有数值都有 `📊` / `📖` / `🔮` 标识
- [ ] 实测数据附环境模板
- [ ] 官方数据有可验证链接
- [ ] 估算数据有置信区间和方法说明
- [ ] 竞品对比使用等价条件
- [ ] 表格底部有统一声明（如包含估算数据）
