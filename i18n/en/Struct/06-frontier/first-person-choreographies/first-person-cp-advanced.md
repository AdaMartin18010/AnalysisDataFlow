## Core Summary

First-Person Choreographic Programming (1CP) is the latest evolution of the Choreographic Programming paradigm, proposed by Graversen et al. in 2025. Unlike traditional third-person Choreography (described from a global perspective), 1CP describes distributed interactions from the participant's own perspective (first-person), supporting runtime dynamic role discovery and process parameterization.

## 1. Definitions

### Def-S-06-1CP-01: First-Person Choreography Formalization

$$
\mathcal{C}_{1CP} ::= (\mathcal{R}, \mathcal{M}, \Sigma, \Pi_{dyn}, \rightsquigarrow)
$$

Where $\Pi_{dyn}$ is the dynamic endpoint projection operator, which decomposes global Choreography into local behavior at runtime (rather than compile time).

### Def-S-06-1CP-02: Continuation-Passing Communication (CPC)

$$
\text{CPC} ::= send(receiver, value, \lambda state. continuation)
$$

### Def-S-06-1CP-03: Census Polymorphism

Allows Choreography to abstract over the number of participants:

$$
\Lambda n. \Lambda \vec{\rho}:\{Role\}^n. \mathcal{C}(\vec{\rho})
$$

## 2. Key Theorems

### Thm-S-06-1CP-01: Dynamic EPP Correctness

The local programs generated by dynamic EPP are compositionally consistent with the global Choreography.

### Thm-S-06-1CP-02: 1CP Completeness

1CP can express all protocols expressible by static Choreography, as well as some dynamic protocols.

## 3. Integration with Stream Processing

1CP is particularly suitable for dynamic stream processing topologies:

- Dynamically adding or removing TaskManagers at runtime
- Adaptive partitioning strategies
- Elastic scaling protocols
