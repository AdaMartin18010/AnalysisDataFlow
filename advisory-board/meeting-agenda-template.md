## 3. 专题评审会议程

### 会议信息

| 项目 | 内容 |
|-----|------|
| **会议名称** | [文档/主题名称] 专题评审会 |
| **日期** | [YYYY-MM-DD] |
| **时间** | [HH:MM - HH:MM] [Timezone] |
| **形式** | 线上 |
| **评审对象** | [文档链接/主题描述] |

### 议程 (总计90分钟)

#### 会前准备

- [ ] 评审材料提前3天发送
- [ ] 顾问独立审阅
- [ ] 收集初步反馈

#### 会议流程

| 时间 | 议题 | 说明 |
|-----|------|------|
| 0:00-0:10 | 背景介绍 | 作者介绍文档背景和目的 |
| 0:10-0:30 | 内容 walkthrough | 作者概述主要内容 |
| 0:30-0:70 | 顾问反馈 | 每位顾问15-20分钟反馈 |
| 0:70-0:85 | 讨论和澄清 | 作者与顾问互动讨论 |
| 0:85-0:90 | 总结和下一步 | 总结反馈，确定修改计划 |

### 评审检查清单

顾问在评审时应关注：

- [ ] **准确性**: 技术细节和数学定义是否准确
- [ ] **完整性**: 是否覆盖了必要的主题
- [ ] **清晰度**: 表达是否清晰易懂
- [ ] **引用**: 引用是否充分且恰当
- [ ] **前沿性**: 是否反映了最新研究进展
- [ ] **客观性**: 对比分析是否公正客观

### 评审反馈表模板

```
文档: [文档名称]
评审人: [姓名]
日期: [日期]

## 总体评价
- [ ] 批准发布
- [ ] 批准发布，建议小修改
- [ ] 需要重大修改后重新评审

## 详细反馈

### 准确性
- 问题1:
- 建议1:

### 完整性
- 建议补充:

### 清晰度
- 改进建议:

### 引用
- 建议添加的引用:

### 其他建议
-

## 优先级建议
- 高优先级修改:
- 中优先级修改:
- 低优先级修改:
```

## 4. 紧急咨询会议

### 适用场景

- 重大技术决策需要快速咨询
- 争议性问题需要第三方意见
- 紧急内容发布前的快速审核

### 会议信息

| 项目 | 内容 |
|-----|------|
| **会议名称** | 紧急咨询: [主题] |
| **日期** | [YYYY-MM-DD] |
| **时间** | [HH:MM - HH:MM] [Timezone] |
| **形式** | 线上/异步 |
| **紧急程度** | [高/中] |

### 议程 (灵活，通常30-60分钟)

1. **问题陈述** (5分钟)
   - 背景说明
   - 需要决策的具体问题

2. **选项展示** (10分钟)
   - 可行方案
   - 各方案利弊

3. **顾问意见** (20-40分钟)
   - 每位顾问发表意见
   - 讨论和澄清

4. **共识/决策** (5-10分钟)
   - 总结意见
   - 形成建议或决策

### 异步咨询模板

如采用异步方式（邮件/GitHub Discussion）：

```
标题: [紧急咨询] [主题]

## 背景
[简要说明背景和紧急性]

## 具体问题
[明确需要咨询的问题]

## 可行选项
1. 选项A: [描述] - 利弊: [说明]
2. 选项B: [描述] - 利弊: [说明]

## 期望反馈
请在 [截止日期] 前回复您的意见：
- 您推荐的选项及理由
- 潜在风险和顾虑
- 其他建议

感谢您的快速响应！
```
