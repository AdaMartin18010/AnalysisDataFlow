## A. Scala 相关术语

### Type Class (类型类)

**定义**: Type Class 是 Scala 中实现**特设多态**（Ad-hoc Polymorphism）的编程模式，通过隐式参数将类型行为与类型定义分离。
形式化定义为接口约束的集合，对于类型 T 满足约束时 T 可获得特定能力。

**相关概念**: Implicits, Higher-Kinded Types, Cats Effect

**来源**: 01.05-scala-type-system-streaming.md

---

### Higher-Kinded Types (HKT)

**定义**: HKT 是类型构造子层面的抽象，允许对类型构造子（如 `List[_]`, `Option[_]`）进行参数化。形式化表示为接受一个类型并返回一个类型的类型算子。

**相关概念**: Type Class, 流类型构造子

**来源**: 01.05-scala-type-system-streaming.md

---

### Implicits (隐式机制)

**定义**: Scala 的隐式参数和隐式转换机制，支持类型类派生、扩展方法和隐式上下文传递。Scala 3 中演进为 `given` 和 `using` 语法。

**相关概念**: Type Class, Extension Methods

**来源**: 01.05-scala-type-system-streaming.md

---

### Macros (宏)

**定义**: Scala 宏是编译期元编程机制，允许在编译阶段生成或转换代码。Scala 2 使用 `macro` 关键字，Scala 3 引入内联和引用/拼接机制。

**相关概念**: Magnolia, Shapeless

**来源**: 01.05-scala-type-system-streaming.md

---

### fs2 (Functional Streams for Scala)

**定义**: 纯函数式流处理库，基于 Cats Effect 实现 Pull-based 流模型。核心抽象为 `Stream[F[_], O]`，其中 F 为效应类型，O 为输出类型。

**相关概念**: Cats Effect, Pekko Streams, Pull-based Backpressure

**来源**: 01.01-scala-streaming-landscape.md

---

### Pekko Streams

**定义**: Apache Pekko 项目中的流处理库，Reactive Streams 规范的实现。提供图构建 DSL（Source -> Flow -> Sink）和背压支持。

**相关概念**: Reactive Streams, Graph DSL, Backpressure

**来源**: 01.04-fs2-pekko-streams.md

---

### Cats Effect

**定义**: Scala 函数式效应系统，提供 `IO` 单子和并发原语（Fiber、Ref、Deferred）。基于 Type Class 设计，支持可测试和可组合的异步编程。

**相关概念**: fs2, ZIO, Fiber

**来源**: 01.01-scala-streaming-landscape.md

---

### ZIO

**定义**: 零依赖的 Scala 效应系统，提供环境 R、错误 E、成功 A 三参数类型 `ZIO[R, E, A]`。内置并发、资源管理和流处理（ZIO Streams）能力。

**相关概念**: ZIO Streams, Cats Effect, Structured Concurrency

**来源**: 01.04-fs2-pekko-streams.md

---

### ZIO Streams

**定义**: ZIO 生态的流处理库，基于 `ZStream[R, E, A]` 类型。特点包括内置背压、信号量控制和与 ZIO 效应的无缝集成。

**相关概念**: ZIO, fs2, Backpressure

**来源**: 01.04-fs2-pekko-streams.md

---

### Magnolia

**定义**: Scala 3 的泛型派生库，用于自动生成 Type Class 实例（如 JSON 编解码器）。基于编译期宏，零运行时依赖。

**相关概念**: Macros, Type Class, Circe

**来源**: 01.05-scala-type-system-streaming.md

---

### Shapeless

**定义**: Scala 2 的泛型编程库，提供 HList（异构列表）、Coproduct 和泛型派生机制。Scala 3 中部分功能由原生 `Mirror` 替代。

**相关概念**: HList, Macros, Type Level Programming

**来源**: 01.05-scala-type-system-streaming.md

---

### Circe

**定义**: Scala 的 JSON 库，基于 Type Class 设计（Encoder, Decoder）。支持自动派生、半自动派生和自定义编解码器。

**相关概念**: Type Class, Magnolia, JSON

**来源**: 01.05-scala-type-system-streaming.md

---

### Extension Methods (扩展方法)

**定义**: Scala 3 语法特性，允许为现有类型添加方法而不修改原类型。使用 `extension` 关键字定义。

**相关概念**: Implicits, Type Class

**来源**: 01.05-scala-type-system-streaming.md

---

### HList (异构列表)

**定义**: Shapeless 库中的核心数据结构，可存储不同类型元素的列表，在类型层面保持元素类型信息。

**相关概念**: Shapeless, Type Level Programming

**来源**: 01.05-scala-type-system-streaming.md

---

### Type Level Programming (类型级编程)

**定义**: 利用 Scala 类型系统进行编译期计算的技术，包括依赖类型、类型级自然数和类型级递归。

**相关概念**: HKT, Shapeless

**来源**: 01.05-scala-type-system-streaming.md

---

### Fiber (纤程)

**定义**: Cats Effect 和 ZIO 中的轻量级并发原语，由运行时调度而非操作系统线程。数万纤程可在少量线程上运行。

**相关概念**: Cats Effect, ZIO, Structured Concurrency

**来源**: 01.04-fs2-pekko-streams.md

---

### Structured Concurrency (结构化并发)

**定义**: 并发编程范式，要求并发任务的生命周期与代码块结构绑定，确保父任务等待子任务完成。

**相关概念**: Fiber, ZIO, Cats Effect

**来源**: 01.04-fs2-pekko-streams.md

---

### Reactive Streams

**定义**: JVM 生态的流处理标准（JSR 366），定义 Publisher、Subscriber、Subscription 和 Processor 接口，规范背压机制。

**相关概念**: Pekko Streams, Backpressure, JDK Flow

**来源**: 01.04-fs2-pekko-streams.md

---

### JDK Flow

**定义**: Java 9 引入的标准 Reactive Streams 实现（java.util.concurrent.Flow），与 Reactive Streams 规范完全兼容。

**相关概念**: Reactive Streams, Pekko Streams

**来源**: 01.04-fs2-pekko-streams.md

---

### Graph DSL

**定义**: Pekko Streams 提供的图构建领域特定语言，支持扇出（Broadcast）、扇入（Merge）和复杂流拓扑的声明式定义。

**相关概念**: Pekko Streams, RunnableGraph

**来源**: 01.04-fs2-pekko-streams.md

---

### RunnableGraph

**定义**: Pekko Streams 中的可执行图抽象，表示封闭的流处理拓扑（所有输入输出都已连接），可执行产生结果。

**相关概念**: Graph DSL, Pekko Streams

**来源**: 01.04-fs2-pekko-streams.md

---

## B. Flink 相关术语

### Checkpoint

**定义**: Flink 的一致性快照机制，基于 Chandy-Lamport 算法，周期性捕获算子状态和数据流位置，用于故障恢复。

**相关概念**: Savepoint, Exactly-Once, Checkpoint Barrier

**来源**: 02.03-flink-state-backends.md

---

### Savepoint

**定义**: 用户触发的 Flink 作业状态快照，与 Checkpoint 语义相同但生命周期由用户管理，用于版本升级、迁移和回滚。

**相关概念**: Checkpoint, State Backend, Job Migration

**来源**: 02.03-flink-state-backends.md

---

### Watermark

**定义**: Flink 中用于标记事件时间进度的特殊元数据，形式为时间戳 t，表示所有时间戳小于 t 的事件已到达（在乱序场景下为大概率保证）。

**相关概念**: Event Time, Window, Allowed Lateness

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Window (窗口)

**定义**: 将无界流切分为有界集合的逻辑分区，支持基于时间（Tumbling、Sliding、Session）或计数的语义。

**相关概念**: Watermark, Trigger, Evictor, Event Time

**来源**: 02.04-flink-sql-table-api.md

---

### State Backend (状态后端)

**定义**: Flink 中负责状态存储和访问的抽象层，决定状态的存储位置（内存/文件系统/RocksDB）和快照机制。

**相关概念**: RocksDB State Backend, ForSt State Backend, Checkpoint

**来源**: 02.03-flink-state-backends.md

---

### ForSt State Backend

**定义**: Flink 2.x 引入的云原生状态后端，实现存算分离架构，状态存储于远程对象存储（S3），本地仅保留缓存。

**相关概念**: State Backend, Disaggregated Compute-Storage, Async State Access

**来源**: 02.03-flink-state-backends.md

---

### RocksDB State Backend

**定义**: 基于 Facebook RocksDB 的嵌入式键值存储后端，状态存储于本地磁盘，支持大状态场景和增量 Checkpoint。

**相关概念**: State Backend, Incremental Checkpoint

**来源**: 02.03-flink-state-backends.md

---

### Operator (算子)

**定义**: Flink 流处理的基本计算单元，将输入流转换为输出流，包括 Source（数据源）、Transformation（变换）和 Sink（数据汇）。

**相关概念**: Operator Chain, Task, Transformation

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Operator Chain (算子链)

**定义**: Flink 将多个兼容算子融合为单个任务执行的优化技术，减少线程切换和序列化开销。

**相关概念**: Operator, Task, Slot Sharing

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Task (任务)

**定义**: Flink 运行时中算子链的物理执行单元，由 TaskManager 中的线程执行。

**相关概念**: Operator, Operator Chain, Task Slot

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Task Slot (任务槽)

**定义**: TaskManager 中资源分配的基本单位，一个 Slot 可执行一个 Task（算子链）。Slot 共享允许同一 Job 的多个 Task 共享 Slot。

**相关概念**: Task, TaskManager, Slot Sharing

**来源**: 02.02-flink-runtime-deep-dive.md

---

### TaskManager (TM)

**定义**: Flink 的工作节点，负责执行具体的数据处理任务。管理一组 Task Slot，与 JobManager 通信接收任务分配。

**相关概念**: JobManager, Task Slot, ResourceManager

**来源**: 02.02-flink-runtime-deep-dive.md

---

### JobManager (JM)

**定义**: Flink 的控制中心，负责任务调度、协调 Checkpoint、故障恢复和资源管理决策。

**相关概念**: TaskManager, ResourceManager, Checkpoint

**来源**: 02.02-flink-runtime-deep-dive.md

---

### ResourceManager

**定义**: Flink 集群的资源管理组件，负责分配和释放 TaskManager 资源，与 Kubernetes、YARN 等资源框架集成。

**相关概念**: JobManager, TaskManager, Adaptive Scheduler

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Exactly-Once (精确一次)

**定义**: 流处理语义保证，每条记录对状态的影响恰好应用一次。Flink 通过 Checkpoint 和幂等 Sink 实现端到端 Exactly-Once。

**相关概念**: At-Least-Once, Checkpoint, Idempotent Sink

**来源**: 02.02-flink-runtime-deep-dive.md

---

### At-Least-Once (至少一次)

**定义**: 流处理语义保证，每条记录至少被处理一次，可能重复。Flink 在部分失败恢复时可能出现重复。

**相关概念**: Exactly-Once, Checkpoint

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Adaptive Scheduler (自适应调度器)

**定义**: Flink 2.x 的动态调度器，根据运行时指标（延迟、背压）自动调整并行度和资源分配。

**相关概念**: Reactive Mode, ResourceManager, JobManager

**来源**: 02.01-flink-2x-architecture.md

---

### Reactive Mode (响应式模式)

**定义**: Flink 2.x 的运行模式，启用 Adaptive Scheduler 的自动扩缩容能力，与 Kubernetes HPA 集成。

**相关概念**: Adaptive Scheduler, Kubernetes, HPA

**来源**: 02.01-flink-2x-architecture.md

---

### Materialized Table (物化表)

**定义**: Flink 2.x Table API/SQL 中的新特性，将流处理结果物化为可查询的存储表，支持增量更新。

**来源**: 02.04-flink-sql-table-api.md

---

### Table API

**定义**: Flink 的关系型 API，提供类型安全的 DSL 定义流/批处理管道，与 SQL 共享执行引擎。

**相关概念**: SQL, Materialized Table, DataStream API

**来源**: 02.04-flink-sql-table-api.md

---

### DataStream API

**定义**: Flink 的核心流处理 API，提供类型安全的数据流操作（map、filter、window、join）。

**相关概念**: Table API, Operator, Transformation

**来源**: 01.02-flink-scala-api-analysis.md

---

### Checkpoint Barrier (检查点屏障)

**定义**: Flink 注入数据流的特殊控制事件，标记 Checkpoint 边界，触发算子状态快照。

**相关概念**: Checkpoint, Chandy-Lamport Algorithm, Exactly-Once

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Chandy-Lamport Algorithm

**定义**: 分布式快照算法，通过标记消息定义一致性全局状态。Flink Checkpoint 基于此算法实现。

**相关概念**: Checkpoint, Checkpoint Barrier, Global Snapshot

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Async State Access (异步状态访问)

**定义**: Flink 2.x 引入的非阻塞状态访问模式，状态 I/O 与计算重叠，提升 CPU 利用率。

**相关概念**: ForSt State Backend, Async Execution

**来源**: 02.01-flink-2x-architecture.md

---

### Async Execution (异步执行)

**定义**: Flink 2.x 的非阻塞执行模型，算子处理逻辑使用回调式异步编程，避免线程阻塞。

**相关概念**: Async State Access, ForSt State Backend

**来源**: 02.01-flink-2x-architecture.md

---

### Disaggregated Compute-Storage (存算分离)

**定义**: 将计算节点与状态存储解耦的架构范式，状态存储于远程对象存储，计算节点无状态。

**相关概念**: ForSt State Backend, Async State Access, Unified File System

**来源**: 02.01-flink-2x-architecture.md

---

### Unified File System (UFS)

**定义**: Flink 的统一文件系统抽象，为存算分离架构提供与 S3、GCS、OSS 等对象存储的适配层。

**相关概念**: Disaggregated Compute-Storage, ForSt State Backend

**来源**: 02.03-flink-state-backends.md

---

### Incremental Checkpoint (增量检查点)

**定义**: 仅持久化自上次 Checkpoint 以来状态变化的优化机制，减少 Checkpoint 时间和存储开销。

**相关概念**: Checkpoint, RocksDB State Backend, State Backend

**来源**: 02.03-flink-state-backends.md

---

### Slot Sharing (Slot 共享)

**定义**: 同一 Job 的多个 Task 共享同一个 Task Slot 的资源优化机制，提高资源利用率。

**相关概念**: Task Slot, Task, Operator Chain

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Transformation (变换)

**定义**: DataStream API 中对流的转换操作（map、filter、flatMap、keyBy、window 等）。

**相关概念**: Operator, DataStream API, KeyBy

**来源**: 01.02-flink-scala-api-analysis.md

---

### KeyBy (按键分组)

**定义**: Flink 的数据分区操作，根据键的哈希值将流数据路由到特定并行子任务，确保同键数据到同一任务。

**相关概念**: Transformation, Key Group, Parallelism

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Key Group (键组)

**定义**: Flink 中键空间的分区单位，每个并行子任务负责一个或多个 Key Group。影响扩缩容时的状态重分布。

**相关概念**: KeyBy, Parallelism, Rescaling

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Parallelism (并行度)

**定义**: Flink 算子的并行实例数量，决定任务在集群中的分布程度。

**相关概念**: Task Slot, Key Group, Adaptive Scheduler

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Rescaling (重扩缩)

**定义**: 动态调整 Flink 作业并行度的操作，通常需要停止作业、创建 Savepoint、恢复时指定新并行度。

**相关概念**: Parallelism, Key Group, Savepoint

**来源**: 02.01-flink-2x-architecture.md

---

### Flink SQL Gateway

**定义**: 提供远程 SQL 执行服务的组件，支持 REST 和 JDBC 协议，使客户端能提交 SQL 查询到 Flink 集群。

**相关概念**: Table API, SQL, JDBC

**来源**: 02.04-flink-sql-table-api.md

## C. Rust 相关术语

### Ownership (所有权)

**定义**: Rust 的核心内存管理机制，每个值有且只有一个所有者，所有者离开作用域时值被自动释放。实现编译期内存安全。

**相关概念**: Borrowing, Lifetimes, RAII

**来源**: 03.01-wasm-interop.md

---

### Borrowing (借用)

**定义**: Rust 中临时访问值而不获取所有权的机制，分为不可变借用（&T）和可变借用（&mut T），遵循借用检查规则。

**相关概念**: Ownership, Lifetimes, Borrow Checker

**来源**: 03.01-wasm-interop.md

---

### Lifetimes (生命周期)

**定义**: Rust 的类型系统特性，标注引用的有效范围，确保引用不会悬垂（dangling）。使用 `'a` 语法表示。

**相关概念**: Ownership, Borrowing, Borrow Checker

**来源**: 03.01-wasm-interop.md

---

### Borrow Checker (借用检查器)

**定义**: Rust 编译器的核心组件，在编译期验证所有权和借用规则，防止数据竞争和内存安全问题。

**相关概念**: Ownership, Borrowing, Lifetimes

**来源**: 03.01-wasm-interop.md

---

### Trait (特质)

**定义**: Rust 中定义类型行为的接口，类似其他语言的接口或 Type Class。支持默认实现、关联类型和泛型约束。

**相关概念**: Generic, Associated Types, Type Class

**来源**: 04.01-rust-engines-comparison.md

---

### Generic (泛型)

**定义**: Rust 的参数化多态机制，允许定义适用于多种类型的代码（如 `Vec<T>`、`Option<T>`）。

**相关概念**: Trait, Associated Types, Type Parameter

**来源**: 04.01-rust-engines-comparison.md

---

### Associated Types (关联类型)

**定义**: Trait 中定义的占位类型，由实现者指定。例如 `Iterator` trait 的 `Item` 类型。

**相关概念**: Trait, Generic, Type Alias

**来源**: 04.01-rust-engines-comparison.md

---

### Async/Await

**定义**: Rust 的异步编程语法糖，`async` 定义异步块/函数，`await` 挂起当前任务等待 Future 完成。基于状态机实现零成本抽象。

**相关概念**: Future, Tokio, Async Runtime

**来源**: 03.01-wasm-interop.md

---

### Future

**定义**: Rust 标准库的异步计算抽象，表示可能尚未完成的计算。`Future::poll` 方法检查计算是否完成。

**相关概念**: Async/Await, Tokio, Executor

**来源**: 03.01-wasm-interop.md

---

### Tokio

**定义**: Rust 生态最流行的异步运行时，提供多线程调度器、异步 I/O、定时器和同步原语。

**相关概念**: Async/Await, Future, Async Runtime

**来源**: 04.01-rust-engines-comparison.md

---

### Async Runtime (异步运行时)

**定义**: 执行 Rust Future 的运行时环境，包含调度器（Scheduler）、Reactor（I/O 事件驱动）和 Timer。

**相关概念**: Tokio, Future, Async/Await

**来源**: 04.01-rust-engines-comparison.md

---

### Executor (执行器)

**定义**: 异步运行时的任务调度组件，负责轮询和唤醒 Future。支持单线程（current_thread）和多线程（multi_thread）模式。

**相关概念**: Tokio, Future, Async Runtime

**来源**: 04.01-rust-engines-comparison.md

---

### unsafe (不安全代码)

**定义**: Rust 中绕过编译器安全检查的代码块，用于底层操作（FFI、原始指针）。开发者需手动保证安全性。

**相关概念**: FFI, Raw Pointer, Miri

**来源**: 03.02-jni-bridge.md

---

### FFI (Foreign Function Interface)

**定义**: Rust 与其他语言（C、Java 等）互操作的机制，使用 `extern` 关键字声明外部函数接口。

**相关概念**: unsafe, JNI, Bindgen

**来源**: 03.02-jni-bridge.md

---

### JNI (Java Native Interface)

**定义**: Java 平台的标准本地接口，允许 JVM 调用本地代码（C/C++/Rust）。Rust 通过 `jni` crate 实现 JNI 绑定。

**相关概念**: FFI, JNI Bridge, unsafe

**来源**: 03.02-jni-bridge.md

---

### Miri

**定义**: Rust 的内存安全检测工具，在解释器层面运行 Rust 代码，检测未定义行为（UB）和内存安全问题。

**相关概念**: unsafe, Undefined Behavior, Sanitizer

**来源**: 03.02-jni-bridge.md

---

### RAII (Resource Acquisition Is Initialization)

**定义**: 资源获取即初始化，Rust 的核心资源管理范式。通过 `Drop` trait 确保资源在对象销毁时释放。

**相关概念**: Ownership, Drop Trait, Scope

**来源**: 03.01-wasm-interop.md

---

### Drop Trait

**定义**: Rust 中定义资源清理行为的 trait，当值离开作用域时自动调用 `drop` 方法。

**相关概念**: RAII, Ownership, Destructor

**来源**: 03.01-wasm-interop.md

---

### Cargo

**定义**: Rust 的构建系统和包管理器，提供依赖管理、构建、测试和发布功能。通过 `Cargo.toml` 配置项目。

**相关概念**: Crate, Rust Toolchain, Crates.io

**来源**: 04.01-rust-engines-comparison.md

---

### Crate (包)

**定义**: Rust 的编译单元，可为二进制（binary）或库（library）。通过 `Cargo.toml` 的 `[dependencies]` 引入依赖。

**相关概念**: Cargo, Module, Crates.io

**来源**: 04.01-rust-engines-comparison.md

---

### Module (模块)

**定义**: Rust 的代码组织机制，使用 `mod` 关键字定义，控制可见性（pub）和命名空间。

**相关概念**: Crate, Visibility, Namespace

**来源**: 04.01-rust-engines-comparison.md

---

### Crates.io

**定义**: Rust 生态的官方包仓库，托管开源 crate。使用 `cargo publish` 发布，`cargo add` 添加依赖。

**相关概念**: Cargo, Crate, Rust Ecosystem

**来源**: 04.01-rust-engines-comparison.md

---

### Rust Toolchain

**定义**: Rust 的编译器工具链，包含 rustc（编译器）、cargo（构建工具）、rustfmt（格式化）、clippy（静态分析）等组件。

**相关概念**: Cargo, rustup, Edition

**来源**: 04.01-rust-engines-comparison.md

---

### rustup

**定义**: Rust 的工具链管理器，用于安装、更新和切换 Rust 版本（stable/beta/nightly）及目标平台。

**相关概念**: Rust Toolchain, Cross Compilation

**来源**: 04.01-rust-engines-comparison.md

---

### Undefined Behavior (UB)

**定义**: 程序行为未由语言规范定义的情况（如空指针解引用、数据竞争），在 unsafe Rust 中可能产生。

**相关概念**: unsafe, Miri, Safety

**来源**: 03.02-jni-bridge.md

### Zero-Cost Abstraction (零成本抽象)

**定义**: Rust 的设计原则，高级抽象在编译后应产生与手写底层代码等价的机器码，无运行时开销。

**来源**: 04.01-rust-engines-comparison.md

---

### fearless Concurrency ( fearless 并发)

**定义**: Rust 的宣传语，指所有权和类型系统确保并发代码的内存安全，无需担心数据竞争。

**相关概念**: Ownership, Send Trait, Sync Trait

**来源**: 04.01-rust-engines-comparison.md

---

### Send Trait

**定义**: 标记可安全跨线程转移所有权的类型，实现 Send 的类型可以在线程间传递。

**相关概念**: Sync Trait, Ownership, Thread Safety

**来源**: 04.01-rust-engines-comparison.md

---

### Sync Trait

**定义**: 标记可安全跨线程共享引用的类型，实现 Sync 的类型可同时被多个线程引用（通过不可变借用）。

## D. WASM/WASI 术语

### WebAssembly (WASM)

**定义**: 一种可移植的二进制指令格式，设计用于在 Web 浏览器中以接近原生的速度运行。现在已扩展到服务器端（WASI）。

**相关概念**: WASI, WASM Module, Host Function

**来源**: 03.01-wasm-interop.md

---

### WASM Module (WASM 模块)

**定义**: WASM 代码的打包单元，包含函数、内存、表和全局变量的定义。扩展名为 `.wasm`。

**相关概念**: WebAssembly, Instance, Host Function

**来源**: 03.01-wasm-interop.md

---

### Host Function (宿主函数)

**定义**: 宿主环境（如 JVM、浏览器）提供给 WASM 模块调用的外部函数，实现 WASM 与宿主的双向通信。

**相关概念**: WebAssembly, WASI, Import/Export

**来源**: 03.01-wasm-interop.md

---

### WASI (WebAssembly System Interface)

**定义**: WebAssembly 的系统接口标准，提供安全、可移植的方式让 WASM 模块与宿主环境交互（文件、网络、时钟等）。

**相关概念**: WebAssembly, WASI 0.3, Component Model

**来源**: 03.01-wasm-interop.md

---

### WASI 0.3

**定义**: WASI 的最新版本（2024+），引入原生异步支持（async/await、stream、future），支持 stack switching 和 continuations。

**相关概念**: WASI, Async I/O, Component Model

**来源**: 03.01-wasm-interop.md

---

### WASI 0.2

**定义**: WASI 的中间版本，引入 Component Model 基础，但异步 I/O 基于轮询（polling）模式。

**相关概念**: WASI, WASI 0.3, Component Model

**来源**: 03.01-wasm-interop.md

---

### Component Model (组件模型)

**定义**: WebAssembly 的高级抽象，允许将多个 WASM 模块组合成可复用组件，通过 WIT 定义跨语言接口。

**相关概念**: WebAssembly, WIT, Lift/Lower

**来源**: 03.01-wasm-interop.md

---

### WIT (WASM Interface Types)

**定义**: WebAssembly 接口类型描述语言，用于定义组件的导入/导出接口，支持跨语言调用。

**相关概念**: Component Model, WebAssembly, wit-bindgen

**来源**: 03.01-wasm-interop.md

---

### wit-bindgen

**定义**: 从 WIT 定义生成语言绑定（Rust、Java、Go 等）的工具，实现跨语言组件调用。

**相关概念**: WIT, Component Model, Language Binding

**来源**: 03.01-wasm-interop.md

---

### Lift (提升)

**定义**: Component Model 中将 WASM 线性内存数据转换为宿主语言对象的操作（Rust -> Scala）。

**相关概念**: Component Model, Lower, WIT

**来源**: 03.01-wasm-interop.md

---

### Lower (降低)

**定义**: Component Model 中将宿主语言对象转换为 WASM 可访问数据的操作（Scala -> Rust）。

**相关概念**: Component Model, Lift, WIT

**来源**: 03.01-wasm-interop.md

---

### Wasmtime

**定义**: Bytecode Alliance 开发的 WASM 运行时，Rust 实现，支持 WASI 和组件模型，提供 C API 和多种语言绑定。

**相关概念**: WASI, WebAssembly, wasmtime-java

**来源**: 03.01-wasm-interop.md

---

### wasmtime-java

**定义**: Wasmtime 的 Java/JVM 绑定，允许在 Scala/Java 应用中嵌入 WASM 运行时。

**相关概念**: Wasmtime, WASI, JNI

**来源**: 03.01-wasm-interop.md

---

### Wasmer

**定义**: 另一个流行的 WASM 运行时，支持多种后端（LLVM、Cranelift、Singlepass），提供多语言 SDK。

**相关概念**: WebAssembly, Wasmtime, Runtime

**来源**: 03.01-wasm-interop.md

---

### wasm3

**定义**: 高性能的 WASM 解释器，适合资源受限环境（嵌入式、IoT），纯 C 实现。

**来源**: 03.01-wasm-interop.md

### wasm32-wasi Target

**相关概念**: WASI, Rust Toolchain, Cross Compilation

**来源**: 03.01-wasm-interop.md

---

### Linear Memory (线性内存)

**定义**: WASM 的内存模型，单块连续的、可增长的内存空间，WASM 模块通过 load/store 指令访问。

### Sandbox (沙箱)

### Capability-based Security (基于能力的权限)

**定义**: WASI 的安全模型，权限不基于用户身份，而基于持有的能力（文件描述符、网络句柄等）。

## E. 流处理通用术语

### Stream Processing (流处理)

**定义**: 对持续产生的无界数据流进行实时计算的范式，与批处理（bounded data）相对。

**相关概念**: Batch Processing, Real-time Processing, Event Stream

**来源**: 01.01-scala-streaming-landscape.md

---

### Event Time (事件时间)

**定义**: 数据记录实际发生的时间（由事件本身携带的时间戳），与 Processing Time（处理时间）相对。

**相关概念**: Processing Time, Watermark, Ingestion Time

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Processing Time (处理时间)

**定义**: 数据记录被流处理系统处理时的机器时间，简单但无法处理乱序事件。

**相关概念**: Event Time, Watermark, Ingestion Time

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Ingestion Time (摄取时间)

**定义**: 数据记录进入流处理系统的时间（由 Source 算子附加的时间戳），介于 Event Time 和 Processing Time 之间。

**相关概念**: Event Time, Processing Time, Source

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Trigger (触发器)

**定义**: 决定窗口计算何时输出的策略，如处理时间触发、事件时间触发（基于 Watermark）或计数触发。

**相关概念**: Window, Watermark, Pane

**来源**: 02.04-flink-sql-table-api.md

---

### Evictor (逐出器)

**定义**: 窗口计算前从窗口中移除元素的策略，用于控制窗口计算时的数据范围。

**相关概念**: Window, Trigger, Allowed Lateness

**来源**: 02.04-flink-sql-table-api.md

---

### Allowed Lateness (允许延迟)

**定义**: 窗口闭合后仍接受迟到事件的时间窗口，超过此时间的事件被丢弃或发送至侧输出。

**相关概念**: Watermark, Window, Late Data

**来源**: 02.04-flink-sql-table-api.md

---

### Late Data (迟到数据)

**定义**: 事件时间小于当前 Watermark 才到达的数据，通常被丢弃或特殊处理。

**相关概念**: Watermark, Allowed Lateness, Side Output

**来源**: 02.04-flink-sql-table-api.md

---

### Side Output (侧输出)

**定义**: Flink 中将不符合主输出条件的数据分流到额外输出流的机制（如迟到数据、异常数据）。

**相关概念**: Late Data, Process Function, Output Tag

**来源**: 02.02-flink-runtime-deep-dive.md

---

### CEP (Complex Event Processing)

**定义**: 复杂事件处理，从事件流中识别符合特定模式的事件序列。Flink CEP 提供模式匹配 API。

**相关概念**: Pattern Matching, Event Sequence, NFA

**来源**: 02.04-flink-sql-table-api.md

---

### Pattern Matching (模式匹配)

**定义**: 在流中识别符合特定规则的事件序列，CEP 的核心能力。支持 Kleene 闭包、时间约束等。

**相关概念**: CEP, Event Sequence, FollowedBy

**来源**: 02.04-flink-sql-table-api.md

---

### Backpressure (背压)

**定义**: 当下游处理速度慢于上游时，流系统自动减速上游生产的机制，防止内存溢出。

**相关概念**: Flow Control, Buffer, Reactive Streams

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Pull-based Backpressure (拉取式背压)

**定义**: 下游主动拉取数据的背压机制（如 fs2），与 Push-based（如 Akka Streams 的信用机制）相对。

**相关概念**: Backpressure, Reactive Streams, fs2

**来源**: 01.01-scala-streaming-landscape.md

---

### Credit-based Flow Control (基于信用的流控)

**定义**: 接收方授予发送方发送配额（credit），发送方在配额内发送数据的流控机制。

**相关概念**: Backpressure, Flink Network Stack, Buffer

**来源**: 02.02-flink-runtime-deep-dive.md

---

### Materialized View (物化视图)

**定义**: 预计算并存储的查询结果，流处理中需增量更新以保持与源数据的一致性。

**相关概念**: Streaming SQL, RisingWave, Incremental Computation

**来源**: 04.02-risingwave-deep-dive.md

---

### Streaming SQL (流式 SQL)

**定义**: 在流数据上执行 SQL 查询的扩展，支持窗口、Watermark、物化视图等流处理语义。

**相关概念**: Materialized View, Window, Flink SQL

**来源**: 02.04-flink-sql-table-api.md

---

### Continuous Query (连续查询)

**定义**: 在流数据上持续运行的 SQL 查询，结果随新数据到达而增量更新。

**相关概念**: Streaming SQL, Materialized View, Incremental View Maintenance

**来源**: 04.02-risingwave-deep-dive.md

---

### Incremental Computation (增量计算)

**定义**: 数据变化时仅重新计算受影响部分的优化技术，流处理的核心优化手段。

**相关概念**: Materialized View, Streaming SQL, Delta

**来源**: 04.02-risingwave-deep-dive.md

---

### Delta (增量)

**定义**: 两次状态之间的差异，增量计算的基本单位。形式化表示为 $\Delta S = S_{new} - S_{old}$。

**相关概念**: Incremental Computation, Differential Dataflow, Materialized View

**来源**: 04.03-materialize-analysis.md

---

### Differential Dataflow

**定义**: 由 Frank McSherry 开发的计算模型，支持任意嵌套循环的增量计算，Materialize 的核心技术。

**相关概念**: Incremental Computation, Materialize, Timely Dataflow

**来源**: 04.03-materialize-analysis.md

---

### Timely Dataflow

**定义**: 微软研究院开发的分布式数据流计算系统，Rust 实现，支持迭代计算和细粒度增量更新。

**相关概念**: Differential Dataflow, Naiad, Rust Streaming

**来源**: 04.01-rust-engines-comparison.md

---

### Nexmark

**定义**: 流处理系统的标准化基准测试套件，模拟在线拍卖场景，包含 23 个标准查询。

**相关概念**: Benchmark, Stream Processing, Performance Testing

**来源**: 04.01-rust-engines-comparison.md

---

### Tumbling Window (滚动窗口)

**定义**: 固定大小、不重叠的时间窗口，如每小时一个窗口。

**相关概念**: Window, Sliding Window, Session Window

**来源**: 02.04-flink-sql-table-api.md

---

### Sliding Window (滑动窗口)

**定义**: 固定大小但可重叠的时间窗口，通过 slide 参数控制窗口间的偏移。

**相关概念**: Window, Tumbling Window, Hop

**来源**: 02.04-flink-sql-table-api.md

---

### Session Window (会话窗口)

**定义**: 根据活动间隙（gap）动态划分的窗口，同一会话内的用户活动被分到同一窗口。

**相关概念**: Window, Tumbling Window, User Activity

**来源**: 02.04-flink-sql-table-api.md

---

### Global Window (全局窗口)

**定义**: 包含所有数据的单一窗口，通常配合自定义 Trigger 使用。

**相关概念**: Window, Trigger, Custom Trigger

**来源**: 02.04-flink-sql-table-api.md

---

### Pane (窗格)

**定义**: 窗口内由 Trigger 触发的子结果，一个窗口可能产生多个 Pane（如处理迟到数据时）。

**相关概念**: Window, Trigger, Late Data

**来源**: 02.04-flink-sql-table-api.md

---

## F. 架构与部署术语

### Kubernetes (K8s)

**定义**: Google 开源的容器编排平台，自动化容器部署、扩展和管理。Flink 原生支持 Kubernetes 部署。

**相关概念**: Docker, Operator, Helm, Container Orchestration

**来源**: 02.05-flink-cloud-native.md

---

### Operator (Kubernetes Operator)

**定义**: 使用自定义资源（CRD）和控制器扩展 Kubernetes 的模式，如 Flink Kubernetes Operator 用于管理 Flink 作业生命周期。

**相关概念**: Kubernetes, CRD, Controller, FlinkDeployment

**来源**: 02.05-flink-cloud-native.md

---

### Helm

**定义**: Kubernetes 的包管理工具，使用 Chart 定义、安装和升级复杂应用。

**相关概念**: Kubernetes, Chart, Release

**来源**: 02.05-flink-cloud-native.md

---

### Docker

**定义**: 容器化平台，将应用及其依赖打包为可移植的容器镜像。

**相关概念**: Kubernetes, Container, Image, Dockerfile

**来源**: 02.05-flink-cloud-native.md

---

### Container (容器)

**定义**: 操作系统级虚拟化技术，隔离进程但共享内核，比虚拟机更轻量。

**相关概念**: Docker, Kubernetes, Image, Namespace

**来源**: 02.05-flink-cloud-native.md

---

### Service Mesh (服务网格)

**定义**: 处理服务间通信的基础设施层，提供流量管理、可观测性和安全，通常通过 Sidecar 代理实现。

**相关概念**: Istio, Envoy, Microservices

**来源**: 05.03-cloud-deployment.md

---

### Istio

**定义**: 流行的开源服务网格实现，基于 Envoy Sidecar，提供流量管理、安全和可观测性。

**相关概念**: Service Mesh, Envoy, Kubernetes

**来源**: 05.03-cloud-deployment.md

---

### Envoy

**定义**: 高性能的 C++ L4/L7 代理，服务网格中常用的数据平面组件。

**相关概念**: Service Mesh, Istio, Proxy

**来源**: 05.03-cloud-deployment.md

---

### Serverless (无服务器)

**定义**: 云计算模型，用户无需管理服务器，按实际使用计费。包括 FaaS（函数即服务）和 BaaS（后端即服务）。

**相关概念**: FaaS, Cloud Functions, AWS Lambda

**来源**: 05.04-edge-computing.md

---

### FaaS (Function as a Service)

**定义**: 函数即服务，事件驱动的无服务器计算模型，用户部署函数代码，平台管理运行时和扩展。

**相关概念**: Serverless, Cloud Functions, Event-driven

**来源**: 05.04-edge-computing.md

---

### Edge Computing (边缘计算)

**定义**: 在数据源附近（边缘节点）进行数据处理的范式，降低延迟和带宽消耗。

**相关概念**: Cloudflare Workers, Arroyo, Low Latency

**来源**: 05.04-edge-computing.md

---

### Cloudflare Workers

**定义**: Cloudflare 的边缘计算平台，基于 V8 Isolate，支持 JavaScript/WASM，在全球 300+ 数据中心运行。

**相关概念**: Edge Computing, Arroyo, V8 Isolate

**来源**: 04.04-arroyo-cloudflare.md

---

### V8 Isolate

**定义**: V8 引擎的轻量级隔离单元，比容器更快启动、更低内存占用，Cloudflare Workers 的执行环境。

**相关概念**: Cloudflare Workers, JavaScript, Sandbox

**来源**: 04.04-arroyo-cloudflare.md

---

### CRD (Custom Resource Definition)

**定义**: Kubernetes 扩展机制，允许用户定义新的 API 资源类型，如 `FlinkDeployment`。

**相关概念**: Kubernetes, Operator, API Server

**来源**: 02.05-flink-cloud-native.md

---

### HPA (Horizontal Pod Autoscaler)

**定义**: Kubernetes 的自动水平扩展机制，根据 CPU、内存或自定义指标调整 Pod 数量。

**相关概念**: Kubernetes, Autoscaling, Reactive Mode

**来源**: 02.05-flink-cloud-native.md

---

### VPA (Vertical Pod Autoscaler)

**定义**: Kubernetes 的自动垂直扩展机制，调整 Pod 的资源请求（CPU/内存）。

**相关概念**: Kubernetes, Autoscaling, Resource Management

**来源**: 02.05-flink-cloud-native.md

---

### Namespace

**定义**: Kubernetes 的资源隔离机制，将集群划分为多个虚拟集群。

**相关概念**: Kubernetes, Resource Quota, RBAC

**来源**: 02.05-flink-cloud-native.md

---

### RBAC (Role-Based Access Control)

**定义**: 基于角色的访问控制，Kubernetes 中通过 Role/ClusterRole 和 RoleBinding 实现权限管理。

**相关概念**: Kubernetes, ServiceAccount, Authorization

**来源**: 02.05-flink-cloud-native.md

---

### ConfigMap

**定义**: Kubernetes 中存储配置数据的对象，可将配置注入容器环境变量或挂载为文件。

**相关概念**: Kubernetes, Secret, Configuration

**来源**: 02.05-flink-cloud-native.md

---

### Secret

**定义**: Kubernetes 中存储敏感数据（密码、Token）的对象，支持加密存储。

**相关概念**: Kubernetes, ConfigMap, Security

**来源**: 02.05-flink-cloud-native.md
