# 🏭 生产案例征集活动

## 活动背景

AnalysisDataFlow 项目致力于打造流计算领域最全面、最严谨的知识库。理论知识与生产实践的结合，是技术价值最大化的关键。

我们诚挚邀请各行业的技术团队，分享您在**流计算生产环境**中的实践经验，帮助更多开发者了解真实场景下的架构设计、问题诊断和优化技巧。

---

## 🎯 征集目标

我们希望通过生产案例征集活动：

1. **建立真实案例库** - 收集不同行业、不同规模的流计算实践案例
2. **促进知识共享** - 让成功经验得到传播，让踩过的坑变成他人的路标
3. **完善项目内容** - 用生产实践反哺理论，让知识库更贴近实际
4. **连接技术社区** - 为流计算从业者提供交流学习的平台

## 📝 提交方式

### 方式一：GitHub Issue（推荐）

1. 访问项目仓库的 [Issues页面](../../issues/new?template=production_case.yml)
2. 选择 "🏭 生产案例提交" 模板
3. 按模板填写案例信息
4. 提交后等待审核

### 方式二：直接提交文档

1. Fork 项目仓库
2. 按照 [templates/production-case-template.md](../templates/production-case-template.md) 模板编写案例
3. 将文档放入 `phase2-case-studies/` 目录
4. 提交 Pull Request

### 方式三：邮件提交

如不便公开提交，可发送邮件至项目维护团队：

- 邮件主题：`[生产案例] {公司名} - {场景名}`
- 附件：案例文档（Markdown格式）
- 内容：是否同意公开、匿名要求等

## ✅ 案例标准

### 内容要求

| 维度 | 要求 | 说明 |
|------|------|------|
| **真实性** | 必须基于真实生产环境 | 不接受虚构或理论推演 |
| **完整性** | 包含背景、架构、挑战、方案、成果 | 参考六段式模板 |
| **技术性** | 有具体的技术细节和实现 | 避免空泛的描述 |
| **可复现** | 关键配置和代码可分享 | 去除敏感信息后可参考 |
| **量化性** | 成果部分包含具体数据 | 性能指标、业务收益等 |

### 审核流程

1. **格式初审**（1-2天）- 检查内容完整性和格式规范
2. **技术评审**（3-5天）- 专家审核技术内容的准确性
3. **内容优化**（双方协作）- 根据反馈完善案例
4. **最终发布** - 合并到主分支并发布公告

详细审核流程参见：[案例审核流程](./case-review-process.md)

## 🔒 隐私保护

我们充分理解企业对信息安全的顾虑：

- **敏感信息脱敏** - 可在案例中隐藏具体公司名称、核心业务数据
- **匿名提交** - 支持以"某电商公司"、"某金融机构"等形式发布
- **内容审核** - 发布前会与您确认最终内容
- **授权协议** - 明确内容使用范围和授权条款

## ❓ 常见问题

**Q: 案例提交后多久能看到？**
A: 一般情况下，从提交到发布需要 1-2 周时间，具体取决于审核反馈的响应速度。

**Q: 可以多次提交不同场景的案例吗？**
A: 非常欢迎！每个不同的技术场景都可以作为独立案例提交。

**Q: 案例中的代码需要开源吗？**
A: 不需要。只需要提供关键配置和代码片段供参考即可，无需完整开源。

**Q: 提交后还能修改吗？**
A: 可以。在审核过程中可以随时更新内容，发布后也可以通过 PR 继续完善。
