## Paths Overview

| Path | Target Audience | Duration | Prerequisites |
|------|-----------------|----------|---------------|
| **Beginner Quick Start** | New to streaming | 2 weeks | Basic programming |
| **Data Engineer** | Building pipelines | 4 weeks | SQL, basic Java/Python |
| **Backend Developer** | Integrating Flink | 6 weeks | Java/Scala, distributed systems |
| **Architect** | Designing systems | 8 weeks | 5+ years distributed systems |
| **Expert Contributor** | Contributing to Flink | Ongoing | Deep JVM, networking, consensus |

## Beginner Quick Start

1. **Week 1: Foundations**
   - Read [Streaming Concepts](FLINK-QUICK-START.md)
   - Understand Event Time vs Processing Time
   - Run first Flink job locally

2. **Week 2: Core Mechanisms**
   - Checkpointing and fault tolerance
   - State backends basics
   - Window operations

## Data Engineer Path

1. **DataStream API** — Transformations, keyed operations, process functions
2. **Table API / SQL** — Declarative stream processing
3. **Connectors** — Kafka, filesystem, JDBC integration
4. **Production** — Monitoring, tuning, deployment patterns
