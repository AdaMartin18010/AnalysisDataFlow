1.1.1 Flink 是什么

Apache Flink 是一个开源的分布式流处理框架，具有以下核心特性：

  • 真正的流处理: 基于事件驱动的微批处理，而非 mini-batch
  • 高吞吐与低延迟: 每秒处理数百万事件，延迟可达毫秒级
  • 精确一次语义: 提供 Exactly-Once 状态一致性保证
  • 事件时间处理: 支持基于事件时间的窗口和 Watermark 机制

1.2.1 程序基本结构

import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.streaming.api.datastream.DataStream;

public class BasicStructure {
    public static void main(String[] args) throws Exception {
        // 1. 创建执行环境
        StreamExecutionEnvironment env = 
            StreamExecutionEnvironment.getExecutionEnvironment();
        
        // 2. 创建数据源 (Source)
        DataStream<String> stream = env.fromElements(
            "Flink", "Spark", "Storm"
        );
        
        // 3. 数据转换 (Transformation)
        DataStream<Integer> lengths = stream
            .map(String::length)
            .filter(len -> len > 4);
        
        // 4. 数据输出 (Sink)
        lengths.print();
        
        // 5. 执行程序
        env.execute("Basic Flink Job");
    }
}

// Keyed Stream 示例: 按用户统计点击次数
DataStream<ClickEvent> clicks = env.addSource(new ClickSource());

clicks
    .keyBy(ClickEvent::getUserId)           // 按用户ID分组
    .map(new RichMapFunction<ClickEvent, UserStats>() {
        private ValueState<Long> countState;
        
        @Override
        public void open(Configuration parameters) {
            countState = getRuntimeContext().getState(
                new ValueStateDescriptor<>("count", Long.class)
            );
        }
        
        @Override
        public UserStats map(ClickEvent event) throws Exception {
            Long current = countState.value();
            if (current == null) current = 0L;
            current++;
            countState.update(current);
            return new UserStats(event.getUserId(), current);
        }
    })
    .print();

1.3.1 WordCount 完整实现

public class StreamingWordCount {
    public static void main(String[] args) throws Exception {
        StreamExecutionEnvironment env = 
            StreamExecutionEnvironment.getExecutionEnvironment();
        
        // 设置并行度
        env.setParallelism(2);
        
        // 从Socket读取数据 (nc -lk 9999)
        DataStream<String> text = env
            .socketTextStream("localhost", 9999);
        
        // WordCount 逻辑
        DataStream<WordCount> wordCounts = text
            .flatMap(new Tokenizer())
            .keyBy(Word::getWord)
            .window(TumblingProcessingTimeWindows.of(Time.seconds(5)))
            .aggregate(new CountAggregate());
        
        wordCounts.print().setParallelism(1);
        
        env.execute("Streaming WordCount");
    }
    
    // 分词器
    static class Tokenizer implements FlatMapFunction<String, Word> {
        @Override
        public void flatMap(String value, Collector<Word> out) {
            for (String word : value.toLowerCase().split("\\W+")) {
                if (word.length() > 0) {
                    out.collect(new Word(word, 1));
                }
            }
        }
    }
}

1.4.1 时间语义对比

时间语义类型:
┌────────────────────────────────────────────────────────┐
│ Event Time    │ 数据产生的时间 (嵌入在记录中)           │ ← 推荐用于生产 │

1.4.2 Watermark 机制

Watermark 是一种特殊的时间戳标记，表示 "所有时间戳小于此值的事件都已到达"。

// 设置时间语义和 Watermark
env.setStreamTimeCharacteristic(TimeCharacteristic.EventTime);

DataStream<Event> withTimestamps = events
    // 方式1: 使用内置策略 (允许5秒乱序)
    .assignTimestampsAndWatermarks(
        WatermarkStrategy
            .<Event>forBoundedOutOfOrderness(Duration.ofSeconds(5))
            .withTimestampAssigner((event, timestamp) -> event.getEventTime())
    );

// 方式2: 自定义 Watermark 生成器
DataStream<Event> withCustomWm = events
    .assignTimestampsAndWatermarks(new CustomWatermarkStrategy());

1.4.3 自定义 Watermark 策略

public class CustomWatermarkStrategy 
    implements WatermarkStrategy<Event> {
    
    @Override
    public WatermarkGenerator<Event> createWatermarkGenerator(
            WatermarkGeneratorSupplier.Context context) {
        return new WatermarkGenerator<Event>() {
            private long maxTimestamp = Long.MIN_VALUE;
            private final long delay = 5000; // 5秒延迟
            
            @Override
            public void onEvent(Event event, long timestamp, WatermarkOutput output) {
                maxTimestamp = Math.max(maxTimestamp, event.getEventTime());
            }
            
            @Override
            public void onPeriodicEmit(WatermarkOutput output) {
                output.emitWatermark(new Watermark(maxTimestamp - delay - 1));
            }
        };
    }
}

1.5.1 窗口类型

窗口类型对比:
┌────────────────────────────────────────────────────────────┐
│ Tumbling Window (滚动窗口)                                  │
│   ├─ 固定大小、不重叠                                        │
│   ├─ 例如: 每5分钟统计一次                                   │
│   └─ [00:00,00:05), [00:05,00:10), [00:10,00:15)...        │
│                                                             │
│ Sliding Window (滑动窗口)                                   │
│   ├─ 固定大小、可重叠                                        │
│   ├─ 例如: 每5分钟计算过去10分钟的统计                       │
│   └─ [00:00,00:10), [00:05,00:15), [00:10,00:20)...        │
│                                                             │
│ Session Window (会话窗口)                                   │
│   ├─ 动态大小、由活动间隙决定                                 │
│   ├─ 例如: 用户30分钟无活动则关闭窗口                         │
│   └─ 适合分析用户行为会话                                    │
└────────────────────────────────────────────────────────────┘

1.5.2 窗口 API 使用

// Tumbling Window 示例
DataStream<Result> tumblingResult = events
    .keyBy(Event::getUserId)
    .window(TumblingEventTimeWindows.of(Time.minutes(5)))
    .aggregate(new AverageAggregate());

// Sliding Window 示例
DataStream<Result> slidingResult = events
    .keyBy(Event::getUserId)
    .window(SlidingEventTimeWindows.of(Time.minutes(10), Time.minutes(5)))
    .aggregate(new CountAggregate());

// Session Window 示例
DataStream<Result> sessionResult = events
    .keyBy(Event::getUserId)
    .window(EventTimeSessionWindows.withGap(Time.minutes(30)))
    .allowedLateness(Time.minutes(5))  // 允许5分钟迟到
    .sideOutputLateData(lateDataTag)    // 迟到数据侧输出
    .aggregate(new SessionAggregate());

1.5.3 窗口函数

// AggregateFunction: 增量聚合 (高效)
public class CountAggregate implements 
    AggregateFunction<Event, Long, Long> {
    
    @Override
    public Long createAccumulator() { return 0L; }
    
    @Override
    public Long add(Event value, Long accumulator) {
        return accumulator + 1;
    }
    
    @Override
    public Long getResult(Long accumulator) { return accumulator; }
    
    @Override
    public Long merge(Long a, Long b) { return a + b; }
}

// WindowFunction: 全量聚合 (可访问窗口元数据)
public class MyWindowFunction implements 
    WindowFunction<Event, Result, String, TimeWindow> {
    
    @Override
    public void apply(String key, TimeWindow window, 
                      Iterable<Event> input, Collector<Result> out) {
        long count = 0;
        for (Event e : input) count++;
        
        out.collect(new Result(
            key, 
            count, 
            window.getStart(), 
            window.getEnd()
        ));
    }
}
