6.1.1 背压原理

背压 (Backpressure) 产生机制:
┌─────────────────────────────────────────────────────────────────┐
│  数据源 ──→ [Fast Operator] ──→ [Slow Operator] ──→ Sink       │
│            (生产快)              (消费慢)                         │
│                 ↑                                               │
│                 └─ 缓冲区填满，向上游传播反压信号                   │
│                                                                 │
│  Credit-based Flow Control:                                     │
│  1. 下游向上游发送 Credit (可用缓冲区数量)                        │
│  2. 上游根据 Credit 发送数据                                      │
│  3. Credit 为 0 时，上游停止发送                                  │
│  4. 反压逐级向上游传播                                            │
└─────────────────────────────────────────────────────────────────┘

背压指标:
- outPoolUsage: 输出缓冲区使用率
- inPoolUsage: 输入缓冲区使用率
- floatingBuffersUsage: 浮动缓冲区使用率
- exclusiveBuffersUsage: 专用缓冲区使用率

6.1.3 背压解决方案

// 方案1: 增加并行度
stream.map(new HeavyComputation())
    .setParallelism(8)  // 增加并行度
    .addSink(new SlowSink());

// 方案2: 异步IO (避免阻塞)
AsyncDataStream.unorderedWait(
    stream,
    new AsyncDatabaseRequest(),  // 异步函数
    1000,  // 超时时间
    TimeUnit.MILLISECONDS,
    100    // 并发请求数
);

// 方案3: 批量处理减少IO
stream.map(new RichMapFunction<Event, Result>() {
    private List<Event> buffer = new ArrayList<>();
    
    @Override
    public Result map(Event event) {
        buffer.add(event);
        if (buffer.size() >= 100) {
            batchProcess(buffer);  // 批量处理
            buffer.clear();
        }
        return null;
    }
});

// 方案4: 增加缓冲区配置
// flink-conf.yaml
taskmanager.memory.network.fraction: 0.15
taskmanager.memory.network.max: 256mb
taskmanager.memory.network.min: 128mb

// 方案5: 拆分重算子
// 将复杂算子拆分为多个简单算子，便于并行和优化
// Before: 一个复杂算子
stream.map(complexFunction);

// After: 拆分后的流水线
stream
    .map(step1)
    .map(step2)
    .map(step3);

6.2.1 Flink 内存模型

Flink TaskManager 内存结构:
┌─────────────────────────────────────────────────────────────────┐
│                         TaskManager 总内存                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    JVM Heap                             │   │
│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │   │
│  │  │  Framework  │  │  Task Slots │  │  Network        │  │   │
│  │  │  Heap       │  │  Heap       │  │  Memory         │  │   │
│  │  │             │  │             │  │                 │  │   │
│  │  └─────────────┘  └─────────────┘  └─────────────────┘  │   │
│  └─────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    Off-Heap Memory                      │   │
│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │   │
│  │  │  Managed    │  │  JVM        │  │  Direct         │  │   │
│  │  │  Memory     │  │  Metaspace  │  │  Memory         │  │   │
│  │  │  (RocksDB)  │  │             │  │                 │  │   │
│  │  └─────────────┘  └─────────────┘  └─────────────────┘  │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘

内存配置参数:
- taskmanager.memory.process.size: 总进程内存
- taskmanager.memory.flink.size: Flink 内存
- taskmanager.memory.managed.fraction: Managed 内存比例
- taskmanager.memory.network.fraction: 网络内存比例

6.3.1 序列化机制对比

Flink 序列化类型 (性能从高到低):
┌─────────────────────────────────────────────────────────────────┐
│ 1. Primitive Types (原生类型)                                    │
│    ├─ int, long, double, boolean, byte[]                        │
│    └─ 直接操作内存，最高效                                       │
│                                                                 │
│ 2. POJO (Plain Old Java Object)                                 │
│    ├─ 满足条件的 Java 类                                         │
│    ├─ 无参构造器、字段可访问、非 final                            │
│    └─ 使用 POJO 序列化器 (字段级序列化)                           │
│                                                                 │
│ 3. Tuple / Row                                                  │
│    ├─ Flink 内置类型                                             │
│    └─ 预编译的序列化器                                           │
│                                                                 │
│ 4. Kryo (通用序列化)                                             │
│    └─ 其他类型的 fallback                                        │
│                                                                 │
│ 5. Avro / Protobuf / Thrift                                     │
│    └─ 外部Schema定义，跨语言支持                                  │
└─────────────────────────────────────────────────────────────────┘

6.3.2 POJO 优化

// 优化前的类 (Kryo 序列化)
public class Event {
    private final String userId;     // final 字段不利于 POJO 识别
    private final long timestamp;
    
    public Event(String userId, long timestamp) {
        this.userId = userId;
        this.timestamp = timestamp;
    }
    // 只有 getter，没有 setter
}

// 优化后的 POJO (Flink 高效序列化)
public class OptimizedEvent {
    public String userId;            // 公共字段或使用 getter/setter
    public long timestamp;
    public int categoryId;
    public double amount;
    
    public OptimizedEvent() {}       // 必须有无参构造器
    
    public OptimizedEvent(String userId, long timestamp,

        this.userId = userId;
        this.timestamp = timestamp;
        this.categoryId = categoryId;
        this.amount = amount;
    }
    
    // 或使用 getter/setter
    public String getUserId() { return userId; }
    public void setUserId(String userId) { this.userId = userId; }
    // ... 其他 getter/setter
}

env.registerType(OptimizedEvent.class);

// 使用示例
stream.map(new RichMapFunction<Event, Result>() {
    private transient Result reusedResult;
    
    @Override
    public void open(Configuration parameters) {
        reusedResult = new Result();
    }
    
    @Override
    public Result map(Event event) {
        // 重用对象，减少GC
        reusedResult.setUserId(event.getUserId());
        reusedResult.setCalculatedValue(compute(event));
        return reusedResult;

            Rule rule = ctx.getBroadcastState(ruleState).get(event.getRuleId());
            // 应用规则
        }
    });

// 5. 分区策略优化

    // 使用 rebalance 均匀分配 (默认 round-robin)

6.4.2 自定义指标

public class MonitoredFunction extends RichMapFunction<Event, Result> {
    private transient Counter eventCounter;
    private transient Histogram processingTimeHistogram;
    private transient Gauge<Long> queueSizeGauge;
    
    private transient Queue<Event> buffer;
    
    @Override
    public void open(Configuration parameters) {
        MetricGroup metricGroup = getRuntimeContext()
            .getMetricGroup()
            .addGroup("custom")
            .addGroup("myFunction");
        
        // 计数器
        eventCounter = metricGroup.counter("eventsProcessed");
        
        // 直方图
        processingTimeHistogram = metricGroup
            .histogram("processingTimeMs", new DropwizardHistogramWrapper(
                new com.codahale.metrics.Histogram(
                    new SlidingWindowReservoir(500)
                )
            ));
        
        // Gauge (瞬时值)
        buffer = new LinkedList<>();
        queueSizeGauge = metricGroup.gauge("bufferSize", () -> (long) buffer.size());
    }
    
    @Override
    public Result map(Event event) {
        long start = System.currentTimeMillis();
        
        // 处理逻辑
        Result result = process(event);
        
        // 记录指标
        eventCounter.inc();
        processingTimeHistogram.update(System.currentTimeMillis() - start);
        
        return result;
    }
}
