实验3: 背压检测实验

可视化背压的传播过程，学习 Credit-based 流控制机制

📖 实验原理

什么是背压 (Backpressure)?

背压是流处理系统中下游处理速度跟不上上游生产速度时产生的压力反向传播现象。Flink 使用 Credit-based 流控制机制来优雅地处理背压。

Credit-based 流控制原理:

📤
发送端
拥有 Credit 才能发送
← Credit 反馈 →
📥
接收端
根据缓冲区余量发放 Credit

📊 背压检测方法

1. Web UI 观察

  • 绿色: 无背压
  • 黄色: 轻度背压
  • 红色: 严重背压

2. 指标监控

  • outPoolUsage: 输出缓冲池使用率
  • backPressuredTimeMsPerSecond
  • numRecordsInPerSecond

3. Flink REST API

GET /jobs/{jobId}/vertices/{id}/backpressure

✏️ 实验任务

任务列表

  1. 观察正常状态下的数据流，记录各阶段的吞吐量
  2. 添加慢速节点，观察背压如何向上游传播
  3. 调整输入速率，找到系统的最大处理能力
  4. 移除瓶颈，观察系统如何恢复正常

💡 背压解决方案

方案1: 增加并行度

提高瓶颈算子的并行度，增加处理能力。适用于计算密集型算子。

方案2: 优化慢速操作

将同步 IO 改为异步 IO (AsyncFunction)，避免阻塞数据流。

方案3: 限流控制

在 Source 端实施限流，控制输入速率匹配处理能力。

方案4: 资源扩容

增加 TaskManager 资源或优化 JVM 参数。
