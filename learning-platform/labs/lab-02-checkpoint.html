实验2: Checkpoint 机制模拟

模拟分布式 Checkpoint 过程，理解 Barrier 传播和状态快照机制

📖 实验原理

Checkpoint 是如何工作的?

Checkpoint 是 Flink 的分布式快照机制，用于故障恢复。其核心是 Chandy-Lamport 算法的变体：

  1. Checkpoint Coordinator 向 Source 发送 Checkpoint 触发指令
  2. Source 注入 Barrier 到数据流中
  3. Barrier 沿数据流传播，算子对齐 Barrier 后进行状态快照
  4. 所有算子完成快照后，Checkpoint 完成

对齐 Checkpoint vs 非对齐 Checkpoint

对齐 Checkpoint: 算子等待所有输入通道的 Barrier 到达后才进行快照。在反压场景下可能导致 Checkpoint 超时。

非对齐 Checkpoint: Barrier 可以超越数据，将积压的数据作为 Checkpoint 的一部分。在反压时仍能快速完成。
